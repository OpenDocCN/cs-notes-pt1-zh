在本节课中，我们将学习如何创建一个 `sendIP` 函数，用于发送原始IP数据包。我们将从定义必要的变量和结构体开始，逐步构建发送逻辑，并最终成功发送一个包含ICMP回显请求的IP数据包。

![](img/2b1eb42b8253bd985e42bf52e49964a4_6.png)

## 概述

我们将创建一个函数来发送原始IP数据包。这涉及到构建IP和ICMP头部，计算数据包总长度，并使用 `sendto` 系统调用将数据包发送到网络。我们还会处理网络字节序等细节问题。

## 构建发送函数

首先，我们需要定义发送函数所需的基本变量和结构体。

我们将需要一个指向数据缓冲区的指针。首先，确认我们拥有所需的一切。如果没有原始套接字，我们需要立即退出。

我们需要一个 `in_addr` 结构体来处理地址。我们还需要一些 `uint16_t` 类型的变量来处理大小等信息。

![](img/2b1eb42b8253bd985e42bf52e49964a4_8.png)

![](img/2b1eb42b8253bd985e42bf52e49964a4_10.png)

我们还需要获取 `sendto` 函数的签名，因为我们可以使用多种不同的系统调用来发送IP数据包。`sendmsg` 要求我们填充一个庞大的结构体，而 `send` 可能更适用于常规的TCP连接。`sendto` 看起来很有希望，因为它允许我们提供文件描述符、包含要发送数据的缓冲区及其长度、标志以及目标地址信息。

## 实现数据包评估与发送

![](img/2b1eb42b8253bd985e42bf52e49964a4_19.png)

接下来，我们需要评估这个IP数据包。我们将创建一个 `eval` 函数来自动处理不同类型的数据包。

![](img/2b1eb42b8253bd985e42bf52e49964a4_21.png)

![](img/2b1eb42b8253bd985e42bf52e49964a4_22.png)

![](img/2b1eb42b8253bd985e42bf52e49964a4_24.png)

`raw` 变量应该等于 `eval` 函数的结果。我们提供数据包结构给 `eval` 函数，它就会准备好要发送的数据包。

然而，我们还需要计算数据包的长度。我们需要为此创建一个函数。数据包的总长度是IP头部结构体的大小加上ICMP结构体的大小，再加上负载的大小。

![](img/2b1eb42b8253bd985e42bf52e49964a4_26.png)

以下是计算大小的代码：
```c
size = sizeof(struct iphdr) + sizeof(struct icmphdr) + payload_size;
```

![](img/2b1eb42b8253bd985e42bf52e49964a4_28.png)

![](img/2b1eb42b8253bd985e42bf52e49964a4_30.png)

![](img/2b1eb42b8253bd985e42bf52e49964a4_32.png)

![](img/2b1eb42b8253bd985e42bf52e49964a4_34.png)

现在，我们有了数据和大小。接下来，我们需要处理 `sendto` 的返回值。`sendto` 通常返回发送的字节数，如果出错则返回 -1。

![](img/2b1eb42b8253bd985e42bf52e49964a4_36.png)

![](img/2b1eb42b8253bd985e42bf52e49964a4_38.png)

我们需要检查返回值是否为 -1。如果发送失败，我们直接返回。

关于 `sendto` 的标志参数，`MSG_DONTWAIT` 表示非阻塞发送。在监听数据包时，我们至少需要使用这个标志，否则无法同时处理多个数据包的发送。但在当前测试阶段，顺序执行所有操作会更简单，所以我们暂时将标志设为 0。

## 修正地址处理

![](img/2b1eb42b8253bd985e42bf52e49964a4_59.png)

我们需要修改 `send` 函数，稍微调整一下。我们需要构造一个 `sockaddr` 结构体。

![](img/2b1eb42b8253bd985e42bf52e49964a4_61.png)

首先清空该结构体。我们只需要提供实际的IP地址。在我们的IP结构体中，目标地址是一个已经是网络字节序的 `in_addr_t`。我们应该能够直接设置它。

![](img/2b1eb42b8253bd985e42bf52e49964a4_63.png)

![](img/2b1eb42b8253bd985e42bf52e49964a4_65.png)

我们需要将我们的IP地址赋值给 `sockaddr_in` 结构体的 `sin_addr` 字段。

![](img/2b1eb42b8253bd985e42bf52e49964a4_67.png)

![](img/2b1eb42b8253bd985e42bf52e49964a4_69.png)

![](img/2b1eb42b8253bd985e42bf52e49964a4_71.png)

## 调试与修正数据包内容

虽然发送成功了，但使用抓包工具查看时，我们发现收到的数据包显示“IP 5 is invalid”。这是一个重要的线索。

![](img/2b1eb42b8253bd985e42bf52e49964a4_77.png)

![](img/2b1eb42b8253bd985e42bf52e49964a4_79.png)

![](img/2b1eb42b8253bd985e42bf52e49964a4_80.png)

查看原始IP数据包，前两个半字节（4位版本号和4位头部长度）的顺序错了。第一个字节是 `0x45`，其中 `4` 是IP版本，`5` 是头部长度（20字节除以4）。但它们目前是 `0x54`，顺序反了。

![](img/2b1eb42b8253bd985e42bf52e49964a4_82.png)

![](img/2b1eb42b8253bd985e42bf52e49964a4_84.png)

这是因为在处理小于一个字节的字段（如4位）时，系统按字节寻址，我们需要考虑字节内的位顺序。我们需要交换这两个4位字段的位置。

![](img/2b1eb42b8253bd985e42bf52e49964a4_86.png)

![](img/2b1eb42b8253bd985e42bf52e49964a4_88.png)

修正IP头部版本和长度的顺序后，重新编译并运行程序。

![](img/2b1eb42b8253bd985e42bf52e49964a4_90.png)

![](img/2b1eb42b8253bd985e42bf52e49964a4_92.png)

成功了！我们现在收到了一个从我们的IP地址到目标主机的ICMP回显请求数据包。标识符（ID）和序列号（Sequence）也显示出来了。

## 完善ICMP负载

![](img/2b1eb42b8253bd985e42bf52e49964a4_94.png)

![](img/2b1eb42b8253bd985e42bf52e49964a4_96.png)

![](img/2b1eb42b8253bd985e42bf52e49964a4_98.png)

![](img/2b1eb42b8253bd985e42bf52e49964a4_100.png)

目前，我们只在ICMP负载中发送了一些垃圾数据。根据标准，Ping的ICMP负载应该包含一个16位的标识符字段和一个16位的序列号字段。

![](img/2b1eb42b8253bd985e42bf52e49964a4_102.png)

我们应该使用这些字段作为负载的一部分。最好的方式是创建一个新的结构体，例如 `ping_payload_t`。

![](img/2b1eb42b8253bd985e42bf52e49964a4_104.png)

这个结构体包含一个16位的 `id` 字段和一个16位的 `seq` 字段，后面可能还会跟着一些数据。

我们需要修改创建数据包的代码。不再使用简单的字符串作为负载，而是使用这个结构体。我们需要正确设置 `id` 和 `seq` 字段，并确保它们是网络字节序。

修改后，我们的负载变成了8个字节（两个16位字段）。再次运行程序，没有错误，并且抓包工具显示我们发送的数据包包含了正确的标识符（5000）和序列号（1）。

![](img/2b1eb42b8253bd985e42bf52e49964a4_106.png)

![](img/2b1eb42b8253bd985e42bf52e49964a4_108.png)
