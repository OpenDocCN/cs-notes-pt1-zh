# 038：创建有效的价值路线图 🗺️

![](img/e7f1bcbc999425607c2dba75f1373e8d_0.png)

在本节课中，我们将学习如何创建一个有效的价值路线图。价值路线图是敏捷项目管理中的关键工具，它能帮助团队规划如何交付最大价值。我们将探讨其核心组成部分，并分享一些实用的创建技巧。

上一节我们介绍了价值路线图的概念及其三个主要组成部分：产品愿景、产品路线图和发布计划。本节中，我们来看看如何将这些部分有效地整合起来，创建一个能指导团队成功交付产品的路线图。

## 创建产品路线图的技巧

产品路线图提供了对预期产品、其需求以及达成里程碑的预估时间线的高层次概览。以下是创建时需要注意的要点。

*   **保持时间预估的灵活性**：产品路线图中的发布日期应仅为粗略估计。在敏捷环境中，变化是常态，尤其是当这些日期可能设定在数月甚至数年之后。过于具体的时间线可能因无法保证而让团队陷入困境。

## 制定发布计划的要点

发布计划连接了产品路线图与团队的实际执行能力。以下是制定发布计划时需要了解的重要事项。

*   **协同制定计划**：产品负责人与项目经理或Scrum主管必须共同制定每个发布计划。这确保了计划能结合产品路线图的目标与团队的**容量**和**速度**（即团队以特定节奏完成工作的能力）。
*   **考虑不可变更的截止日期**：如果路线图中存在任何“硬性截止日期”（即不可更改的日期），必须将其纳入可能受影响的发布计划中。例如，公司可能计划在十月的某个行业大会上推出服务的第一阶段。
*   **明确沟通“必须有”的功能**：与利益相关者清晰沟通硬性截止日期，以便团队明确哪些是“必须有”的核心功能。这样，当团队发现可能无法按时完成所有任务时，可以迅速聚焦于这些核心功能。
*   **将发布计划视为“活的”文件**：拥有发布计划并不意味着抗拒变化。敏捷团队将发布计划视为一个“活的”产物，可以根据环境变化和接收到的新信息进行调整。

## 可能导致发布计划变更的因素

既然发布计划需要保持灵活，那么了解哪些因素可能触发变更就很重要。以下是几个常见的因素。

*   **团队速度的变化**：团队在给定迭代或冲刺中能完成的工作量可能发生变化。原因可能是团队成员的增加或减少，甚至是工作效率的提升。
*   **产品范围的变更**：如果产品负责人批准了对产品范围的更改。
*   **对功能所需工作量的理解加深**：团队在深入研究或更好地理解产品领域后，可能会发现某个用户故事或史诗比最初预想的更简单或更复杂。

## 发布计划的持续审查

我的最后一个建议是，Scrum主管或项目经理在开始每次冲刺规划会议前，都应审查发布计划。

![](img/e7f1bcbc999425607c2dba75f1373e8d_2.png)

他们通过审查来检查团队是否按计划推进。如果团队偏离了轨道，Scrum主管需要与产品负责人及相关业务人员开诚布公地讨论，找出可以调整的地方以重回正轨。这正是Scrum价值观中 **“透明”** 的关键所在。

一个有效的价值路线图是构建和交付成功产品的强大工具。您所制定的计划将帮助团队专注于交付最大价值，同时保持灵活性和敏捷性。
