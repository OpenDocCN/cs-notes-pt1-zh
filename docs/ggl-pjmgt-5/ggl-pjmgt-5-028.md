# 028：每日站会的益处 🗣️

![](img/cc245d25901fe616c69c0ffa6c842e36_0.png)

在本节课中，我们将学习敏捷项目管理中的一个核心实践——每日站会。我们将了解它的目的、如何高效组织，以及它如何促进团队协作与项目透明度。

大家好，我是Sarah，是谷歌工程教育团队的项目经理。

我认为，有一种集中和组织信息的方式不像文档那样被经常提及，那就是与你的跨职能项目团队举行每日站会或每周站会。

有时作为项目经理，你可能会倾向于认为，我们的数据分析师不需要知道营销人员在做什么，因为他们的工作截然不同。

但事实上，给予每个人每周参加一次会议的机会，快速交流一下“我刚完成了什么”、“我正在做什么”，这能创造一定程度的可见性和透明度。

这种方式还能培养一种团队精神和协作感，而仅仅记录所有事情却不进行交流，是无法真正实现这一点的。

站会的目标不是开成像回顾会或全员大会那样冗长的会议。

## 站会的核心原则 ⏱️

![](img/cc245d25901fe616c69c0ffa6c842e36_2.png)

上一节我们提到了站会的目标，本节中我们来看看组织站会需要遵循哪些核心原则。

![](img/cc245d25901fe616c69c0ffa6c842e36_3.png)

这些会议应尽可能简短，我通常将其安排在15或20分钟，如果更短我会更高兴。

我们真正希望的是人们能快速说明他们当前的工作内容以及是否有任何阻碍。

任何可能阻碍特定队友或项目某部分取得进展的问题。

那么你可能需要与那位特定人员或工作流负责人安排额外的时间来处理这些任务，看看你能做些什么来帮助他们摆脱困境。

## 考虑团队分布与时间安排 🌍

在理解了站会的基本原则后，我们需要考虑实际执行中的挑战，特别是当团队分布在不同地区时。

关于每日站会的另一件需要考虑的事情是，你很可能会与许多不同时区、不同地点的人一起工作。

我建议发送一份快速调查，询问大家“上午11点（某时区）可以吗？”或“下午4点（某时区）可以吗？”，然后在将其列入日历并期望每个人每次都出席之前，确保这感觉像是达成了共识。

如果你正在处理一个截止日期非常紧迫的项目，并且每天都要开站会，我尽量避免营造一种强制要求每个人每天都必须出席的文化。

项目的某些部分可能在某些时候进展得更快。

## 高效主持会议的技巧 📝

组织会议只是第一步，如何主持一场高效的站会同样关键。以下是确保会议聚焦且富有成效的一些方法。

一个让人们在时间盒内、在每日站会或任何类型的会议中保持专注的好方法是：在会议前创建议程，并实际在旁边写下你希望为每个更新分配多少分钟。

但如果出现了计划外的情况，我通常不会中途打断他们。

与你的利益相关者或项目团队合作时，直接说“两分钟到了，你说完了”绝不是一种好的工作方式。

但可以在谈话出现自然停顿的某个时刻说：嘿，我们实际上需要给另一件事再留出10到15分钟，所以我们能否先把这个问题放一放，如果稍后有时间再回来讨论，或者留到下次会议。

本节课中我们一起学习了每日站会的核心价值与组织技巧。我们了解到，站会旨在通过简短、频繁的同步，提升团队工作的可见性、透明度和协作感。关键在于控制时长、包容不同时区、提前制定议程并灵活处理讨论，从而确保会议高效且对团队成员有价值。
