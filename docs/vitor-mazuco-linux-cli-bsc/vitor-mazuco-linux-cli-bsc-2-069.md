# 069：创建强密码策略 🔐

在本节课中，我们将学习如何在MySQL数据库中创建并启用强密码策略。数据库安全至关重要，而设置强密码是防止未授权访问的第一道防线。我们将通过启用密码验证插件，并配置具体的密码规则来实现这一目标。

上一节我们介绍了MySQL的基本安全概念，本节中我们来看看如何具体实施密码策略。

## 启用密码验证插件

![](img/3a9f0757cae87683816451c22941583d_1.png)

![](img/3a9f0757cae87683816451c22941583d_3.png)

![](img/3a9f0757cae87683816451c22941583d_4.png)

MySQL提供了多种插件来增强功能，密码验证插件就是其中之一。首先，我们需要找到并启用它。

![](img/3a9f0757cae87683816451c22941583d_6.png)

![](img/3a9f0757cae87683816451c22941583d_7.png)

你可以通过以下SQL命令查看插件目录的位置：
```sql
SELECT @@plugin_dir;
```

![](img/3a9f0757cae87683816451c22941583d_9.png)

![](img/3a9f0757cae87683816451c22941583d_11.png)

![](img/3a9f0757cae87683816451c22941583d_13.png)

在我的Ubuntu Linux服务器上，插件目录位于`/usr/lib/mysql/plugin/`。该目录下存放着许多可用的插件，例如用于密码验证、连接控制等的插件。

![](img/3a9f0757cae87683816451c22941583d_15.png)

我们将启用密码验证相关的插件。以下是操作步骤：

1.  编辑MySQL的配置文件（通常是`/etc/mysql/my.cnf`或`/etc/my.cnf`）。
2.  在`[mysqld]`部分添加或修改以下行来启用插件：
    ```
    plugin-load-add=validate_password.so
    ```
3.  保存配置文件并重启MySQL服务以使更改生效：
    ```bash
    sudo systemctl restart mysql
    ```

## 配置密码策略规则

![](img/3a9f0757cae87683816451c22941583d_17.png)

插件启用后，我们可以配置具体的密码强度要求。这些规则对普通用户生效，超级用户（如root）通常不受限制。

![](img/3a9f0757cae87683816451c22941583d_19.png)

![](img/3a9f0757cae87683816451c22941583d_21.png)

我们可以通过以下SQL命令查看当前的密码策略设置：
```sql
SHOW VARIABLES LIKE 'validate_password%';
```

初始策略可能强度不足。我们可以将其配置为要求密码至少包含以下元素：
*   **长度**：至少10个字符。
*   **大小写字母**：混合大写和小写字母。
*   **数字**：至少包含2个数字。
*   **特殊字符**：至少包含1个非字母数字的特殊字符（如`%`, `$`, `#`）。

![](img/3a9f0757cae87683816451c22941583d_23.png)

你可以根据安全需求调整这些值，例如增加最小长度或特殊字符的数量。

![](img/3a9f0757cae87683816451c22941583d_25.png)

![](img/3a9f0757cae87683816451c22941583d_26.png)

配置完成后，记得再次重启MySQL服务。

![](img/3a9f0757cae87683816451c22941583d_28.png)

## 测试密码策略

![](img/3a9f0757cae87683816451c22941583d_30.png)

现在，让我们验证策略是否生效。我们将尝试为一个普通用户（例如`application_user`）修改密码。

![](img/3a9f0757cae87683816451c22941583d_32.png)

首先，使用一个弱密码进行测试：
```sql
SET PASSWORD FOR 'application_user'@'localhost' = PASSWORD('weakpass');
```
由于密码不符合我们设置的策略（长度不足、缺少数字和特殊字符等），此命令将会失败。

接下来，尝试使用一个符合所有规则的强密码：
```sql
SET PASSWORD FOR 'application_user'@'localhost' = PASSWORD('Str0ngP@ssw0rd!');
```
这个密码包含了大小写字母、数字和特殊字符，并且长度足够，因此应该能够成功设置。

## 使用函数验证密码强度

![](img/3a9f0757cae87683816451c22941583d_34.png)

MySQL还提供了一个`VALIDATE_PASSWORD_STRENGTH()`函数，可以在设置密码前评估其强度。该函数返回一个0到100的分数，分数越高代表密码越强。

![](img/3a9f0757cae87683816451c22941583d_36.png)

![](img/3a9f0757cae87683816451c22941583d_38.png)

以下是几个示例：
```sql
SELECT VALIDATE_PASSWORD_STRENGTH('abc'); -- 返回0，表示非常弱
SELECT VALIDATE_PASSWORD_STRENGTH('MediumPass1'); -- 返回约50，表示中等强度
SELECT VALIDATE_PASSWORD_STRENGTH('V3ry$tr0ngP@ss!'); -- 返回100，表示非常强
```
在为用户创建密码前，可以先使用此函数进行验证，确保其达到所需的强度标准。

![](img/3a9f0757cae87683816451c22941583d_40.png)

本节课中我们一起学习了如何通过启用和配置MySQL的密码验证插件来实施强密码策略。我们了解了如何设置具体的密码规则（如长度、字符类型），如何测试策略是否生效，以及如何使用内置函数评估密码强度。为数据库用户设置强密码是保障数据安全的一项基本且必要的措施。
