Linux命令行基础：Part2：Linux性能调优基础原则 🚀

在本节课中，我们将学习如何优化Linux内核性能。我们将探讨`/proc`目录的作用，学习如何查看和修改内核参数，并了解进行系统调优时必须遵循的重要安全原则。

---

### 概述 📋

Linux内核本身非常稳定，但通过一些调整，我们可以进一步提升系统的日常性能。然而，内核调优需要格外谨慎，任何微小的错误都可能导致系统无法正常工作。因此，在开始之前，我们必须明确一些基本原则。

---

### 核心原则与安全须知 ⚠️

在进行任何内核优化之前，请务必遵守以下规则：

以下是必须遵守的核心安全原则：

1.  **禁止在生产服务器上测试配置**：这听起来是常识，但很多人会犯这个错误。永远不要在正在运行的生产系统上直接修改内核参数。
2.  **在非生产环境充分测试**：所有的配置更改都应在测试系统上完成。在应用到生产环境前，至少进行三次完整的测试。
3.  **制定详细的计划**：在开始修改前，明确你想要达到的优化目标。一个周密的计划可以防止生产系统发生灾难性故障。

遵循这些原则是安全进行内核调优的第一步。

### 探索内核信息中心：`/proc` 目录 🔍

![](img/f8cb37094d03beb37dc5920f0860e4b8_5.png)

![](img/f8cb37094d03beb37dc5920f0860e4b8_7.png)

现在，让我们进入实践环节。所有内核相关的信息和可调参数都位于 `/proc` 目录中。

执行命令 `cd /proc` 进入该目录。`/proc` 是一个虚拟文件系统，它提供了访问内核内部数据和状态的接口。

![](img/f8cb37094d03beb37dc5920f0860e4b8_9.png)

![](img/f8cb37094d03beb37dc5920f0860e4b8_11.png)

如果在此目录执行 `ls` 命令，你会看到许多条目。其中，蓝色的数字目录代表当前系统正在运行的**进程ID（PID）**。每个进程都有一个唯一的PID，你可以进入相应的PID目录（例如 `cd /947`）查看该进程的详细信息，如状态、内存使用情况和CPU限制等。

![](img/f8cb37094d03beb37dc5920f0860e4b8_13.png)

![](img/f8cb37094d03beb37dc5920f0860e4b8_15.png)

![](img/f8cb37094d03beb37dc5920f0860e4b8_16.png)

除了进程信息，`/proc` 目录还包含许多重要的系统文件：

*   **`cmdline`**：显示启动当前Linux内核时使用的命令行参数。
*   **`meminfo`**：提供系统内存使用的详细信息。
*   **`cpuinfo`**：显示CPU的详细信息。
*   **`partitions`**：列出所有存储设备和分区。
*   **`modules`**：显示所有已加载的内核模块。
*   **`sys/`**：这是进行内核调优的关键目录，包含了大量可调整的参数。

`/proc` 目录是了解和优化Linux性能的核心入口。

---

### 理解可调参数：`/proc/sys/` 目录 ⚙️

性能调优的主要操作在 `/proc/sys/` 目录下进行。这个目录包含了内核的众多可调参数，它们被组织成不同的子目录：

*   **`abi/`**：应用程序二进制接口，通常不需要修改。
*   **`crypto/`**：加密相关设置，修改复杂且不常见。
*   **`debug/`**：调试目录，可以启用或禁用，但不常用。
*   **`dev/`**：用于设备调整。
*   **`fs/`**：虚拟文件系统接口。例如，`file-max` 参数控制可以同时打开的最大文件数。
*   **`kernel/`**：Linux内核本身的大量可调参数。
*   **`net/`**：网络相关配置。
*   **`sunrpc/`**：NFS文件系统相关。
*   **`user/`**：用户相关设置。
*   **`vm/`**：虚拟内存管理接口。

每个子目录下都有具体的文件，对应着不同的系统参数。

### 修改内核参数的方法 🛠️

![](img/f8cb37094d03beb37dc5920f0860e4b8_27.png)

![](img/f8cb37094d03beb37dc5920f0860e4b8_28.png)

![](img/f8cb37094d03beb37dc5920f0860e4b8_30.png)

![](img/f8cb37094d03beb37dc5920f0860e4b8_32.png)

有两种主要命令可以修改 `/proc/sys/` 下的参数：

![](img/f8cb37094d03beb37dc5920f0860e4b8_34.png)

![](img/f8cb37094d03beb37dc5920f0860e4b8_36.png)

1.  **使用 `echo` 命令**：直接将值写入参数文件。
    ```bash
    echo <value> > /proc/sys/<path/to/parameter>
    ```
2.  **使用 `sysctl` 命令**：这是专门用于配置内核参数的工具。
    *   查看所有变量：`sysctl -a`
    *   修改参数：`sysctl -w <parameter>=<value>`

![](img/f8cb37094d03beb37dc5920f0860e4b8_38.png)

![](img/f8cb37094d03beb37dc5920f0860e4b8_40.png)

![](img/f8cb37094d03beb37dc5920f0860e4b8_41.png)

例如，我们可以简单地修改主机名：
```bash
# 查看当前主机名
cat /proc/sys/kernel/hostname

![](img/f8cb37094d03beb37dc5920f0860e4b8_43.png)

![](img/f8cb37094d03beb37dc5920f0860e4b8_44.png)

# 使用 echo 修改主机名
echo "new-hostname" > /proc/sys/kernel/hostname
```

但请注意，使用 `echo` 或 `sysctl -w` 进行的修改是**临时**的，系统重启后会失效。

### 实战：永久修改内核参数 💡

![](img/f8cb37094d03beb37dc5920f0860e4b8_46.png)

![](img/f8cb37094d03beb37dc5920f0860e4b8_47.png)

为了让配置在重启后依然生效，我们需要创建永久的配置文件。

![](img/f8cb37094d03beb37dc5920f0860e4b8_49.png)

![](img/f8cb37094d03beb37dc5920f0860e4b8_51.png)

让我们以一个常见的网络安全配置为例：**禁用ICMP Ping响应**（使他人无法ping通你的服务器）。

![](img/f8cb37094d03beb37dc5920f0860e4b8_53.png)

1.  **检查当前状态**：首先，ping一下本机，确认是通的。
    ```bash
    ping localhost
    ```

![](img/f8cb37094d03beb37dc5920f0860e4b8_55.png)

2.  **找到并临时修改参数**：控制Ping响应的参数是 `/proc/sys/net/ipv4/icmp_echo_ignore_all`。值为 `0` 表示允许Ping，`1` 表示忽略。
    ```bash
    # 查看当前值
    cat /proc/sys/net/ipv4/icmp_echo_ignore_all

    # 临时禁用Ping (设为1)
    echo 1 > /proc/sys/net/ipv4/icmp_echo_ignore_all
    # 或使用 sysctl
    sysctl -w net.ipv4.icmp_echo_ignore_all=1
    ```
    再次执行 `ping localhost`，会发现请求超时，修改生效。

3.  **创建永久配置**：临时修改在重启后会恢复。要永久生效，需要在 `/etc/sysctl.d/` 目录下创建一个配置文件（例如 `99-custom.conf`）。
    ```bash
    # 编辑配置文件
    sudo vim /etc/sysctl.d/99-custom.conf
    ```
    在文件中添加以下行：
    ```
    net.ipv4.icmp_echo_ignore_all = 1
    ```
    保存并退出。

4.  **使永久配置生效**：重启系统，或者立即加载所有配置文件：
    ```bash
    sudo sysctl --system
    ```
    现在，即使重启服务器，Ping功能也保持被禁用的状态。

这个例子演示了从查看、临时测试到永久修改一个内核参数的标准流程。

### 总结 🎯

本节课我们一起学习了Linux性能调优的基础：

1.  **安全第一**：永远不在生产环境直接测试，制定计划，充分测试。
2.  **信息中心**：`/proc` 目录是获取内核和进程信息的宝库。
3.  **调优核心**：`/proc/sys/` 目录包含了所有可调整的内核参数。
4.  **修改方法**：可以使用 `echo` 命令或 `sysctl` 工具来修改参数。
5.  **永久生效**：通过向 `/etc/sysctl.d/` 目录添加 `.conf` 配置文件，可以使更改在系统重启后持续有效。

![](img/f8cb37094d03beb37dc5920f0860e4b8_57.png)

记住，内核调优是一个需要耐心和谨慎的过程。每次修改前，务必理解其含义，并在测试环境中反复验证，确保不会对系统稳定性造成影响。
