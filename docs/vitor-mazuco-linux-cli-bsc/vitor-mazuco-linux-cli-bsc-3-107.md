# 107：二元操作符第二部分 🔢

在本节课中，我们将要学习PromQL中二元操作符的更多用法，特别是如何将它们应用于瞬时向量和标量，以进行更复杂和有趣的数据分析。

上一节我们介绍了二元操作符的基本概念，本节中我们来看看如何将它们应用于不同类型的数据。

## 标量与瞬时向量

除了操作符，我们还需要理解瞬时向量和标量。标量是那些可以写成数字后跟测量单位的形式的数值，例如1米、1分钟、200克、5摄氏度。所有这些我们使用的测量值都称为标量。

我们可以将标量用于算术运算，以改变瞬时向量中的值。

以下是使用内存指标的一个例子：

```promql
process_resident_memory_bytes / 1024
```

这个查询将内存使用量从字节转换为千字节，使数据更易于阅读和理解。这个除法操作被应用于Prometheus中的所有时间序列。

## 算术操作符

我们拥有多种算术操作符。

以下是主要的算术操作符列表：
*   **加法**：`+`
*   **减法**：`-`
*   **乘法**：`*`
*   **除法**：`/`
*   **取模**：`%`
*   **幂运算**：`^`

取模运算符（`%`）是浮点取模，如果提供非整数输入，它可以返回非整数结果。例如，`5 % 1.5` 的计算结果是一个标量值。

## 操作符的应用方式

二元算术操作符可以在两种操作数都是标量时使用。这对于提高数据的可读性非常有用。正如我们之前所做的，将内存指标转换为千字节或兆字节比使用原始值更容易理解。

我们也可以将标量操作数放在操作符的左侧，将瞬时向量放在右侧。

例如：
```promql
1e9 - process_resident_memory_bytes
```
这个查询从10亿（1e9）中减去内存使用量。

## 比较操作符

我们还有比较操作符，用于测试条件。

以下是主要的比较操作符列表：
*   **大于**：`>`
*   **小于**：`<`
*   **等于**：`==`
*   **大于等于**：`>=`
*   **小于等于**：`<=`
*   **不等于**：`!=`

我们可以使用瞬时向量与标量进行比较。例如，查询 `process_cpu_seconds_total > 10` 会过滤出值大于10的指标（如Prometheus进程），而 `process_cpu_seconds_total < 10` 则会过滤出值小于10的指标（如Node进程）。通过组合这些操作符，可以进行更精细的数据筛选。
