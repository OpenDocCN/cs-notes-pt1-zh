# 113：索引介绍 📚

在本节课中，我们将要学习MongoDB中索引的基础概念。索引是提升数据库查询性能的关键技术，理解其工作原理对于高效使用数据库至关重要。

## 什么是索引？

索引与书籍或PDF文件末尾的索引非常相似。当你想在书中查找某个内容时，你可以选择逐页翻阅整本书，也可以直接查阅索引。索引能让你更快地定位到目标内容所在的页码。

在数据库中，索引的作用完全相同。如果没有索引，MongoDB需要扫描集合中的所有文档来查找匹配项。而通过索引，数据库可以创建一种特殊的数据结构，它存储了数据的一个小子集，并以一种更易于定位和遍历的方式组织起来，从而支持更高效的搜索。

## 索引如何工作？

索引会按照指定的顺序，存储特定字段或一组字段的值。这个顺序由创建索引时定义的字段值决定。

![](img/db3f976b9cdf9b510748e5811a81f04b_1.png)

例如，如果我们想在MongoDB中创建一个索引，语法非常简单。你需要指定数据库、集合名，然后使用 `createIndex` 方法。你需要提供要建立索引的键（字段名），以及可选的排序方向参数。

**代码示例：创建单字段索引**
```javascript
db.collection.createIndex({ title: 1 })
```
这里的 `key` 是你想要创建索引的字段名（例如 `title`）。`1` 代表升序排列。如果你想使用降序，只需使用负数 `-1` 即可。数字序列（1,2,3,4...）会一直持续到你的硬盘无法容纳为止。

**代码示例：创建多字段复合索引**
```javascript
db.collection.createIndex({ title: 1, description: -1 })
```
如果你想创建包含多个字段的索引，只需用逗号分隔各个字段及其排序方向即可。例如，可以创建一个 `title` 字段升序、`description` 字段降序的复合索引。之后，你就可以在查询字段中使用这些索引方法了。

## 索引的可选参数

MongoDB的 `createIndex` 方法提供了一些非常有用的可选参数。以下是其中几个关键的参数：

*   **background**: 此参数允许索引在后台构建，不会阻塞MongoDB的其他操作。
*   **unique**: 此参数创建一个唯一索引，确保集合中该索引字段的值是唯一的，不接受重复文档的插入。
*   **name**: 此参数用于指定索引的名称。如果未指定，MongoDB会自动生成一个名称，通常是索引字段名和排序方向的拼接。
*   **sparse**: 此参数用于创建稀疏索引。我们将在下一节课中重点讨论这个参数。
*   **partialFilterExpression**: 此参数允许你使用过滤表达式来创建部分索引。
*   **storageEngine**: 此参数用于指定存储引擎选项，目前使用较少。

在接下来的课程中，我们将重点实践稀疏索引，并探讨为什么以及何时应该使用这种类型的索引。
