# 125：分析数据库与集合 📊

在本节课中，我们将学习如何分析 MongoDB 数据库及其集合的存储情况。我们将通过导入一个包含邮政编码的 CSV 文件到 MongoDB 中，然后使用特定命令来查看数据库和集合的详细统计信息，例如数据大小、索引数量等。这对于数据库管理员预估存储需求和规划服务器资源至关重要。

## 导入数据到 MongoDB

上一节我们介绍了 MongoDB 的基本操作，本节中我们来看看如何将外部数据导入到数据库中。首先，我们需要下载一个包含邮政编码的 CSV 文件。

以下是下载和导入文件的步骤：

1.  使用 `wget` 命令下载 CSV 文件。
2.  使用 `ls` 命令确认文件已成功下载。
3.  使用 `mongoimport` 命令将 CSV 文件中的数据导入到指定的 MongoDB 数据库和集合中。

`mongoimport` 命令的基本格式如下：
```bash
mongoimport --db <数据库名> --collection <集合名> --type csv --file <文件名.csv> --headerline
```
如果数据库启用了认证，你还需要在命令中添加用户名、密码和认证数据库的参数。

执行导入命令后，系统会建立连接。如果目标集合已存在，它会被删除并重新创建。命令会报告导入的文档数量，例如 3,939,000 个文档。

## 验证数据与创建索引

数据导入后，我们需要进入 MongoDB 环境进行验证。

以下是验证数据和创建索引的步骤：

1.  使用 `use test` 命令切换到 `test` 数据库。
2.  使用 `db.pincodes.count()` 或 `db.pincodes.findOne()` 命令来确认数据已成功导入。
3.  为邮政编码字段创建一个唯一索引，以确保数据的唯一性并提升查询效率。

创建唯一索引的命令如下：
```javascript
db.pincodes.createIndex({ pincode: 1 }, { unique: true })
```
默认情况下，MongoDB 会为每个文档自动生成一个 `_id` 字段并为其建立索引。因此，集合现在拥有两个索引：默认的 `_id` 索引和我们刚创建的 `pincode` 索引。

## 查看集合统计信息

现在数据已经准备就绪，我们可以开始分析集合的详细情况。查看集合统计信息的核心命令是 `db.collection.stats()`。

使用以下命令查看 `pincodes` 集合的统计信息：
```javascript
db.pincodes.stats()
```
该命令会返回一个包含多种统计数据的文档。默认情况下，大小的单位是字节（Bytes）。为了更直观地阅读，我们可以指定缩放因子（scale），将字节转换为千字节（KB）或兆字节（MB）。

例如，要查看以 KB 为单位的大小信息，可以运行：
```javascript
db.pincodes.stats(1024) // 结果以KB显示
```
要查看以 MB 为单位的大小信息，可以运行：
```javascript
db.pincodes.stats(1024 * 1024) // 结果以MB显示
```
返回的统计信息包括：
*   **`size`**：集合中所有文档的总大小。
*   **`count`**：集合中的文档总数。
*   **`avgObjSize`**：每个文档的平均大小。
*   **`storageSize`**：为存储集合数据而分配的空间大小。
*   **`nindexes`**：集合上索引的总数。
*   **`totalIndexSize`**：所有索引的总大小。
*   **`indexSizes`**：详细列出每个索引的大小。

通过比较不同缩放因子下的结果，我们可以更清晰地了解集合在磁盘上的实际占用情况。

## 查看数据库统计信息

分析完集合后，我们还可以查看整个数据库的概况。查看数据库统计信息的命令是 `db.stats()`。

使用以下命令查看当前数据库（例如 `test`）的统计信息：
```javascript
db.stats()
```
同样，你也可以使用缩放因子来调整显示单位：
```javascript
db.stats(1024 * 1024) // 结果以MB显示
```
该命令返回的信息包括：
*   **`db`**：数据库名称。
*   **`collections`**：数据库中的集合数量。
*   **`views`**：数据库中的视图数量。
*   **`objects`**：数据库中所有文档的总数。
*   **`avgObjSize`**：所有文档的平均大小。
*   **`dataSize`**：数据的总大小。
*   **`storageSize`**：存储空间的总大小。
*   **`indexes`**：索引的总数。
*   **`indexSize`**：所有索引的总大小。

这些统计数据对于评估数据库的整体资源消耗和进行容量规划非常有帮助。
