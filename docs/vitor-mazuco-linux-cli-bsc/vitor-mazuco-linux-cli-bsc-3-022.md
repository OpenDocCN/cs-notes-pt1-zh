# 022：blob、tree与commit 🧱🌳💾

## 概述
在本节课中，我们将学习Git版本控制系统的三个核心对象：blob、tree和commit。我们将通过实际操作，了解如何创建这些对象，并理解它们在Git仓库中的角色和工作流程。

---

上一节我们介绍了Git的基本概念，本节中我们来看看如何具体创建和管理Git的内部对象。

首先，请确保你已经下载了在前几节课中创建的Git仓库。我们将在这个仓库上进行操作。当你使用`ls`命令时，可能只看到几个文件，但关键的配置信息都存储在名为`.git`的隐藏目录中。这个目录包含了仓库的所有设置、提交记录和分支信息。

让我们回到主目录，并对一个现有文件进行简单的修改。

当我们使用`git add`命令时，文件从工作目录被移动到暂存区。在这个过程中，Git会为文件内容创建一个**blob**对象。Blob是“二进制大对象”的缩写，它被记录在Git的数据库中。每次添加文件，都会生成一个唯一的SHA-1哈希值作为该blob的ID。

![](img/dd7a04e0ba93d4992940157193f7315b_31.png)

这看起来可能有些抽象，但其目的是为了高效存储文件内容。即使你之后删除了不需要的文件，Git数据库中也已经保存了必要对象的记录。

![](img/dd7a04e0ba93d4992940157193f7315b_33.png)

现在，让我们再次编辑同一个文件。

```bash
echo "第二次修改的内容" >> another_file
```

![](img/dd7a04e0ba93d4992940157193f7315b_35.png)

![](img/dd7a04e0ba93d4992940157193f7315b_37.png)

编辑后，运行`git status`，你会看到有趣的现象：文件同时出现在了两个部分。

![](img/dd7a04e0ba93d4992940157193f7315b_39.png)

![](img/dd7a04e0ba93d4992940157193f7315b_41.png)

![](img/dd7a04e0ba93d4992940157193f7315b_18.png)

![](img/dd7a04e0ba93d4992940157193f7315b_43.png)

绿色部分显示的是已暂存的修改（即第一次添加的内容A），而红色部分显示的是尚未暂存的新修改。这是因为第一次添加时，文件内容A被哈希并存储到了Git数据库中。而第二次的修改还没有被哈希和记录。

![](img/dd7a04e0ba93d4992940157193f7315b_45.png)

所以，我们目前的状态是：一部分更改已准备就绪可以提交（暂存区），另一部分更改还未被跟踪（工作区）。如果我们再次运行`git add another_file`，那么所有最新的更改都会被添加到暂存区，状态将恢复为“更改已准备提交”。

![](img/dd7a04e0ba93d4992940157193f7315b_46.png)

![](img/dd7a04e0ba93d4992940157193f7315b_48.png)

最后，我们将暂存区的更改创建为一个**commit**（提交）对象。

![](img/dd7a04e0ba93d4992940157193f7315b_50.png)

以下是提交更改的命令：
```bash
git commit -m “测试更改消息”
```

![](img/dd7a04e0ba93d4992940157193f7315b_52.png)

提交操作会创建一个**commit**对象，该对象包含了作者信息、时间戳、指向当前**tree**对象的指针以及指向父提交的指针（如果是首次提交则没有父提交）。Tree对象则记录了当前项目目录的结构和所有blob的引用。

![](img/dd7a04e0ba93d4992940157193f7315b_54.png)

如果系统提示需要配置用户邮箱，你可以使用以下命令进行配置：
```bash
git config --global user.email "你的邮箱"
```

![](img/dd7a04e0ba93d4992940157193f7315b_56.png)

提交完成后，我们可以查看创建的对象。使用`git cat-file -p <提交哈希>`可以查看commit对象的详细信息，包括其关联的tree。再通过tree对象，就能看到它引用的所有blob和文件名。

此外，`git fsck`命令可以检查仓库中未被任何引用指向的“悬空对象”。

![](img/dd7a04e0ba93d4992940157193f7315b_58.png)

![](img/dd7a04e0ba93d4992940157193f7315b_60.png)

如果你想查看详细的提交历史，可以使用`git log`命令。

## 总结
本节课中我们一起学习了Git的三个核心对象：**blob**、**tree**和**commit**。我们通过修改文件、添加到暂存区并最终提交更改的完整流程，实践了这些对象的创建过程。理解这些底层对象有助于我们更深入地掌握Git的工作原理。
