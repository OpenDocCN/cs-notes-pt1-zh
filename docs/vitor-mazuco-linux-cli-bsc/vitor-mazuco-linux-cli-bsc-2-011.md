在本节课中，我们将简要介绍什么是通用安全策略。安全策略是描述组织内每个人必须遵循的规则和实践的定义，无论组织规模大小，其核心目标是保护公司的数据和信息。

通用安全策略是一套规则，用于指导组织内所有成员在配置和维护计算机网络安全时应遵循的实践。它旨在保护公司的机密数据和敏感信息。

以下是制定安全策略时必须涵盖的一些核心主题：
*   **系统访问权限**：明确谁可以访问系统。
*   **软件安装与维护权限**：确定谁有权安装、修改或维护软件及系统（包括从路由器到大型服务器的各种设备）。
*   **数据权限与访问**：规定谁有权访问哪些数据。
*   **故障恢复责任**：如果发生故障，明确由谁负责执行恢复操作（例如，根据备份策略进行数据恢复）。

创建安全策略时，应遵循**最小权限原则**，即用户只能使用被明确授予的服务和权限，公司内部的许多操作必须受到限制。这虽然有时显得繁琐，但至关重要。

接下来，让我们从一个众所周知的简单话题开始：密码保护。

![](img/0d85cf8ca3efbf184fc2c47ecf942546_73.png)

![](img/0d85cf8ca3efbf184fc2c47ecf942546_75.png)

![](img/0d85cf8ca3efbf184fc2c47ecf942546_76.png)

密码是任何系统中非常重要的安全因素。一个不安全的密码可能导致公司内部信息被破坏。因此，制定良好的密码保护策略需要公司全体成员，从普通用户到高级安全专家，共同参与。

![](img/0d85cf8ca3efbf184fc2c47ecf942546_78.png)

![](img/0d85cf8ca3efbf184fc2c47ecf942546_80.png)

以下是一些保护密码安全的实用建议：
*   **避免密码复用**：用户不应在公司所有账户上使用相同的密码。个人网站、应用程序、社交媒体账户与公司账户的密码应完全不同。
*   **分级使用不同密码**：不同安全级别的账户必须使用不同的密码。
*   **严格保密**：密码应被视为敏感机密信息，**绝不**通过电话、电子邮件、短信或任何其他方式告知或分享给任何人，包括同事、经理甚至家人。
*   **安全存储**：避免将密码写在纸上或存储在未加密的移动设备中。如需存储，应使用具有强加密功能的专业密码管理应用。
*   **及时报告与更改**：如果怀疑密码可能已泄露，应立即报告安全事件，并更改所有相关密码。许多大公司在发生数据泄露后，都会要求用户重置密码。
*   **设置强密码规范**：密码必须符合一定的强度标准，不能是随意设置的。一个强密码通常需要满足以下条件：
    `密码长度 >= 8个字符`，并且组合使用**大写字母**、**小写字母**、**数字**和**特殊字符**（如 !, @, #），同时避免使用姓名、生日等易猜信息。
*   **定期审计**：对于大型公司，拥有专门团队进行安全审计是必不可少的。该团队需要检查密码强度、查看安全报告，并评估系统本身的安全性。

服务器遭受恶意攻击的一个主要原因是存在未修复的漏洞。如果管理员没有正确安装系统或遵循安全最佳实践，就会使系统处于脆弱状态。因此，无论是设置服务器、台式机还是任何其他系统，都必须严格遵循正确的安全配置流程。

![](img/0d85cf8ca3efbf184fc2c47ecf942546_96.png)

以下是一些服务器安全的通用策略：
*   **明确管理责任**：内部服务器的管理必须有一个专门的团队全权负责。所有相关事务，包括合规性检查和事件响应，都应由该团队承担。
*   **资产登记与识别**：所有内部服务器都必须进行登记，以便识别。记录信息应包括其物理位置、运行的软件版本、操作系统类型以及硬件配置。
*   **持续更新与打补丁**：所有应用程序和服务必须始终保持更新。因为所有系统都可能存在漏洞。当发现重要漏洞时，管理员必须及时知晓并快速修复，从简单的WiFi路由器固件到大型操作系统，无一例外。
*   **保持警惕与持续投入**：安全维护是持续的过程，并非一劳永逸。公司需要保持警惕，持续投入。这需要一支合格的团队，并对全体员工（而不仅仅是技术人员）进行持续培训。

![](img/0d85cf8ca3efbf184fc2c47ecf942546_100.png)

所有这些措施都必须与公司组织内的每个人充分协调，以尽可能避免出现严重的安全问题或事件。
