**MongoDB 入门：第2章：使用 TTL 索引实现文档自动过期 🗑️⏱️**

在本节课中，我们将学习 MongoDB 中一个非常实用的功能：TTL（生存时间）索引。这个功能允许我们为集合中的文档设置一个“过期时间”，MongoDB 会自动在指定时间后删除这些文档。这对于管理临时数据（如用户会话、验证码或日志）非常有用。

---

### **TTL 索引的工作原理**

TTL 索引是一种特殊的单字段索引。MongoDB 会启动一个后台线程，持续检查被索引字段的值。当系统时间超过该字段值指定的时间点时，对应的文档就会被自动删除。

其核心是一个简单的**公式**：
`删除时间 = 文档中的日期字段值 + 指定的过期秒数`

或者，更直接地说，当 `当前服务器时间 > 文档的日期字段值` 时，文档过期并被删除。

---

### **方法一：固定过期时间**

首先，我们来看如何让集合中的所有文档都在创建后固定的时间（例如5分钟）后过期。

上一节我们介绍了TTL索引的基本概念，本节中我们来看看具体的实现方法。以下是创建固定过期时间TTL索引的步骤：

1.  **准备数据与函数**：我们需要一个JavaScript函数来创建测试数据和索引。假设我们有一个名为 `addTTLTestData` 的函数。
2.  **创建测试文档**：函数会先删除已存在的 `ttl_test` 集合，然后插入三个文档。每个文档包含一个 `createdAt` 字段，其值被设置为不同的过去时间（例如当前时间减去2、3、4分钟），这是为了模拟文档在不同时间点被创建。
3.  **创建TTL索引**：这是关键步骤。我们在 `createdAt` 字段上创建一个索引，并指定 `expireAfterSeconds` 选项为300秒（5分钟）。

创建索引的**代码**命令如下：
```javascript
db.ttl_test.createIndex({ “createdAt“: 1 }, { expireAfterSeconds: 300 })
```
这行代码意味着：以 `createdAt` 字段创建索引，并且任何在此集合中的文档，将在其 `createdAt` 字段记录的时间点之后 **300秒** 被自动删除。

4.  **观察效果**：执行函数并创建索引后，你可以立即查询集合，会看到三个文档。由于它们的 `createdAt` 时间都是过去时，并且已经超过了5分钟，所以MongoDB会很快（通常在1分钟内）将它们全部删除。使用 `db.ttl_test.find()` 命令可以观察到文档逐个消失的过程。

这种方法适用于所有文档需要统一生命周期的情况。

---

### **方法二：自定义过期时间**

![](img/8e7e4ce7036689db7cbe627800860b92_1.png)

现在，我们来看看更灵活的方式：如何让每个文档拥有自己独立的过期时间，而不是统一的固定时长。

上一节我们实现了固定时间的过期，但有时我们需要更精细的控制。例如，某些文档可能1小时后过期，而另一些可能需要1天后过期。以下是实现方法：

1.  **修改文档结构**：我们不再依赖一个固定的 `expireAfterSeconds` 值。相反，我们在每个文档中直接存储一个明确的过期时间点。通常，这个字段可以命名为 `expireAt`。
2.  **创建TTL索引**：在 `expireAt` 字段上创建TTL索引，但这次将 `expireAfterSeconds` 设置为 **0**。

创建索引的**代码**命令如下：
```javascript
db.ttl_test_2.createIndex({ “expireAt“: 1 }, { expireAfterSeconds: 0 })
```
`expireAfterSeconds: 0` 表示：当服务器时间**超过**文档 `expireAt` 字段所存储的具体时间点时，立即删除该文档。

3.  **插入文档时指定过期时间**：现在，当你插入文档时，需要为每个文档计算并设置 `expireAt` 字段的值。

例如，创建一个5分钟后过期的文档，其**代码**如下：
```javascript
db.ttl_test_2.insertOne({
    “data“: “临时会话“,
    “expireAt“: new Date(Date.now() + 5 * 60 * 1000) // 当前时间 + 5分钟
})
```
你可以自由地为每个文档设置不同的 `expireAt` 值，实现高度个性化的过期策略。

---

### **总结**

本节课中我们一起学习了 MongoDB TTL 索引的两种应用方式：
*   **固定过期时间**：通过在创建索引时设置 `expireAfterSeconds` 参数，让集合内所有文档遵循统一的过期时长。关键在于索引的配置。
*   **自定义过期时间**：通过将 `expireAfterSeconds` 设为 `0`，并在每个文档中设置一个 `expireAt` 字段来指定精确的过期时间点。这提供了更灵活的、基于文档级别的过期控制。

TTL 索引是自动化数据管理的强大工具，能有效帮助清理临时数据，保持数据库的简洁和高效。你可以根据实际业务场景，选择最适合的方案。
