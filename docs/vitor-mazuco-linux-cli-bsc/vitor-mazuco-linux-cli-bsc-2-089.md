## 数组

上一节我们介绍了MongoDB的基本概念，本节中我们来看看数组类型。数组是一种可以有序存储多个值的结构，类似于列表或电子表格中的一行。

![](img/1b74f54145ee014345d26df7b7b0d6dd_6.png)

在MongoDB中，一个键可以对应一个数组值。这意味着你可以为单个键存储一组值，例如一个列表、集合或栈。这允许你在一个字段中创建多种类型的值，只要它们属于同一个键。

![](img/1b74f54145ee014345d26df7b7b0d6dd_8.png)

![](img/1b74f54145ee014345d26df7b7b0d6dd_9.png)

以下是数组在文档中的一个示例：
```json
{
  "tags": ["mongodb", "database", "nosql"]
}
```

## 嵌入式文档

![](img/1b74f54145ee014345d26df7b7b0d6dd_14.png)

![](img/1b74f54145ee014345d26df7b7b0d6dd_16.png)

接下来，我们探讨嵌入式文档。我们已经学习过文档，但MongoDB还允许将文档嵌套在其他文档内部，这为数据结构提供了更大的复杂性。

![](img/1b74f54145ee014345d26df7b7b0d6dd_18.png)

![](img/1b74f54145ee014345d26df7b7b0d6dd_19.png)

例如，一个用户文档中可以包含一个完整的地址文档作为其值。这种设计方式与SQL数据库不同，它能让你更好地利用数据库的结构。

![](img/1b74f54145ee014345d26df7b7b0d6dd_21.png)

以下是一个嵌入式文档的示例：
```json
{
  "name": "Victor Hugo",
  "address": {
    "street": "123 Main St",
    "city": "Anytown",
    "zip": "12345"
  }
}
```
在这个例子中，`address`是键，其值是一个包含`street`、`city`和`zip`键的完整文档。

![](img/1b74f54145ee014345d26df7b7b0d6dd_22.png)

## 对象ID

![](img/1b74f54145ee014345d26df7b7b0d6dd_24.png)

![](img/1b74f54145ee014345d26df7b7b0d6dd_25.png)

最后，我们介绍对象ID。在MongoDB中，当你创建一个文档时，系统会自动生成一个唯一的`_id`字段，即对象ID。

![](img/1b74f54145ee014345d26df7b7b0d6dd_27.png)

对象ID相当于传统关系型数据库（如MySQL、PostgreSQL）中的主键，用于在集合中唯一标识一个文档。它由12个字节组成，遵循特定的生成规则：

![](img/1b74f54145ee014345d26df7b7b0d6dd_28.png)

![](img/1b74f54145ee014345d26df7b7b0d6dd_30.png)

以下是对象ID的构成部分：
1.  前4个字节：代表Unix时间戳（自1970年1月1日以来的秒数）。
2.  接着的3个字节：代表机器标识符。
3.  随后的2个字节：代表进程ID。
4.  最后的3个字节：是一个以随机值开始的递增计数器。

![](img/1b74f54145ee014345d26df7b7b0d6dd_32.png)

这种结构确保了即使在多台机器上同时生成ID，也能保持全局唯一性。一个典型的对象ID看起来像这样：`ObjectId("507f1f77bcf86cd799439011")`。

---

![](img/1b74f54145ee014345d26df7b7b0d6dd_34.png)

本节课中我们一起学习了MongoDB中的三种核心数据类型：**数组**、**嵌入式文档**和**对象ID**。理解这些类型有助于你设计更灵活、高效的数据库结构。数组允许存储值列表，嵌入式文档支持复杂的嵌套数据，而对象ID则为每个文档提供了唯一标识。掌握这些概念是使用MongoDB进行有效数据建模的基础。
