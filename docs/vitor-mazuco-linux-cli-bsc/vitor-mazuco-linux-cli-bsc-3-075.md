# 075：创建基本拨号计划用于测试第二部分

![](img/0bd048652914db57cd6347bdc0d80714_1.png)

## 概述
在本节课中，我们将学习如何在Asterisk中创建和配置一个基本的拨号计划。我们将从备份现有配置文件开始，逐步创建一个简单的拨号上下文，并探索多种配置方法，包括使用宏和正则表达式来优化配置。通过本教程，你将能够建立一个可以实际工作的拨号环境。

---

## 备份现有配置文件
在开始配置之前，我们首先需要备份现有的配置文件，以防止数据丢失。

以下是备份步骤：
1.  进入Asterisk的配置目录：`cd /etc/asterisk`
2.  使用`mv`命令重命名`extensions.conf`文件以创建备份：`mv extensions.conf extensions.conf.backup`
3.  使用`touch`命令重新创建一个空的配置文件：`touch extensions.conf`

备份完成后，我们就可以在一个干净的文件中开始配置了。

---

## 理解拨号计划结构
上一节我们备份了配置文件，本节中我们来看看拨号计划的基本结构。一个拨号计划主要由上下文（contexts）、分机（extensions）和优先级（priorities）组成。

![](img/0bd048652914db57cd6347bdc0d80714_3.png)

![](img/0bd048652914db57cd6347bdc0d80714_5.png)

### 全局参数 [general]
`[general]`部分用于设置全局参数，这些参数通常用于宏定义或一些无需在其他部分重复配置的通用设置。目前我们可以暂时将其留空。

```ini
[general]
```

### 创建拨号上下文
上下文定义了拨号规则的作用域。每个分支（如分机）都可以有自己的上下文。为了标准化和易于识别，我们创建的上下文名称最好具有明确的含义。

![](img/0bd048652914db57cd6347bdc0d80714_7.png)

我们创建一个名为`branches`的上下文：

```ini
[branches]
```

![](img/0bd048652914db57cd6347bdc0d80714_9.png)

## 配置基本分机
现在，让我们在`[branches]`上下文中配置一个最简单的分机。我们将配置分机1002，使其能够呼叫另一个PJSIP分机。

以下是分机1002的配置代码：
```ini
exten => 1002,1,Dial(PJSIP/1002)
exten => 1002,2,Answer()
exten => 1002,3,Hangup()
```

**代码解析：**
*   `exten => 1002,1,...`：这定义了分机号码`1002`的第一个动作（优先级1）。
*   `Dial(PJSIP/1002)`：这是拨号函数，表示呼叫PJSIP协议的分机`1002`。
*   `exten => 1002,2,Answer()`：这是第二个动作（优先级2），用于接听来电。
*   `exten => 1002,3,Hangup()`：这是第三个动作（优先级3），使用`Hangup()`函数挂断通话。

配置完成后，保存文件。每次修改拨号计划后，都需要在Asterisk CLI中重载配置使其生效：

![](img/0bd048652914db57cd6347bdc0d80714_11.png)

```bash
asterisk -rx “dialplan reload”
```

你可以使用`asterisk -rx “dialplan show branches”`命令来查看我们刚创建的上下文和分机配置。

## 测试拨号连接
配置重载后，就可以进行测试了。确保你的SIP电话（如软电话或硬件话机）已注册到Asterisk服务器，并且分机`1002`已正确配置。

![](img/0bd048652914db57cd6347bdc0d80714_15.png)

测试步骤：
1.  使用一个分机（例如1001）拨打`1002`。
2.  分机`1002`应振铃，接听后即可进行通话。
3.  挂断后，通话结束。

> **注意**：VoIP通话质量非常依赖于网络状况。请确保使用稳定的网络连接，建议至少使用4G或有线网络，并选择合适的编解码器（如G.711、G.729）以减少延迟和提升通话质量。

## 优化配置方法
我们的基本拨号计划已经可以工作，但Asterisk提供了更简洁的配置方式。让我们来优化一下代码。

![](img/0bd048652914db57cd6347bdc0d80714_21.png)

### 方法一：使用相同的分机模式
我们可以省略重复的分机号，系统会自动按顺序分配优先级。

![](img/0bd048652914db57cd6347bdc0d80714_23.png)

![](img/0bd048652914db57cd6347bdc0d80714_25.png)

优化后的配置如下：
```ini
exten => 1002,1,Dial(PJSIP/1002)
same => n,Answer()
same => n,Hangup()
```
这里的`same => n`表示“与上一行相同的分机，下一个优先级”。这样写更加简洁。

### 方法二：使用模式匹配
如果我们有一系列分机（如1000-1009），可以使用模式匹配来简化配置。使用下划线`_`和`X`（代表0-9的任意数字）来定义模式。

![](img/0bd048652914db57cd6347bdc0d80714_27.png)

以下是匹配1000到1009的配置：
```ini
exten => _100X,1,Dial(PJSIP/${EXTEN})
same => n,Answer()
same => n,Hangup()
```
*   `_100X`：匹配1000、1001、1002……1009。
*   `${EXTEN}`：这是一个Asterisk宏，它代表实际拨打的号码。如果拨打1002，那么`${EXTEN}`的值就是1002。

### 方法三：使用`${EXTEN}`宏
这是最灵活和推荐的方法，特别适用于分机数量较多的情况。它直接使用拨打的号码作为呼叫目标。

![](img/0bd048652914db57cd6347bdc0d80714_29.png)

配置代码如下：
```ini
exten => _1XXX,1,Dial(PJSIP/${EXTEN})
same => n,Answer()
same => n,Hangup()
```
*   `_1XXX`：匹配所有以1开头的四位分机号（如1000-1999）。
*   无论拨打哪个符合模式的分机号（如1003、1500），系统都会尝试呼叫对应的PJSIP分机。

![](img/0bd048652914db57cd6347bdc0d80714_31.png)

![](img/0bd048652914db57cd6347bdc0d80714_33.png)

> **提示**：如果被呼叫的分机不存在，Asterisk会返回“无法接通”的提示音或消息。

![](img/0bd048652914db57cd6347bdc0d80714_35.png)

## 添加语音提示功能
除了呼叫分机，拨号计划还可以播放语音提示。例如，我们可以设置一个分机，当被呼叫时播放“Hello World”的音频文件。

以下是配置一个播放语音提示的分机2000的示例：
```ini
exten => 2000,1,Answer()
same => n,Playback(hello-world)
same => n,Hangup()
```
*   `Playback(hello-world)`：函数用于播放名为`hello-world`的音频文件（该文件通常位于`/var/lib/asterisk/sounds/`目录下）。

![](img/0bd048652914db57cd6347bdc0d80714_39.png)

保存并重载拨号计划后，拨打分机2000，即可听到“Hello World”的语音提示。

## 总结
本节课中我们一起学习了如何构建一个Asterisk拨号计划。我们从备份配置文件开始，逐步创建了一个包含上下文和基本分机的拨号环境。我们深入探讨了多种配置语法：
*   基础的分机和优先级定义。
*   使用`same => n`优化连续动作。
*   利用模式匹配（如`_100X`）来覆盖多个分机号。
*   通过`${EXTEN}`宏实现动态呼叫。
*   最后，我们还学习了如何使用`Playback()`函数来提供语音提示。

![](img/0bd048652914db57cd6347bdc0d80714_45.png)

通过这些步骤，你已经创建了一个简单但功能完整的拨号计划，并掌握了对其进行优化和扩展的基本方法。你可以在此基础上，继续探索更复杂的Asterisk功能，如IVR（交互式语音应答）、呼叫队列等。
