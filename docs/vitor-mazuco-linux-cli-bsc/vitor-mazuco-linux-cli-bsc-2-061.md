# 061：处理数值数据

在本节课中，我们将学习MySQL中用于处理数值数据的不同数据类型。我们将重点了解整数类型、定点数类型及其存储特性、范围和应用场景。

## 整数类型

![](img/a4f337962ef1c0294751807526423018_1.png)

上一节我们介绍了数值数据的基本概念，本节中我们来看看具体的整数类型。整数类型用于存储没有小数部分的精确数值。

![](img/a4f337962ef1c0294751807526423018_3.png)

![](img/a4f337962ef1c0294751807526423018_5.png)

以下是MySQL中主要的整数类型及其存储范围：

![](img/a4f337962ef1c0294751807526423018_7.png)

*   **TINYINT**：存储空间为1字节。有符号范围：-128 到 127。无符号范围：0 到 255。
*   **SMALLINT**：存储空间为2字节。有符号范围：-32768 到 32767。无符号范围：0 到 65535。
*   **MEDIUMINT**：存储空间为3字节。有符号范围：-8388608 到 8388607。无符号范围：0 到 16777215。
*   **INT** 或 **INTEGER**：存储空间为4字节。有符号范围：-2147483648 到 2147483647。无符号范围：0 到 4294967295。
*   **BIGINT**：存储空间为8字节。有符号范围：-2^63 到 2^63-1。无符号范围：0 到 2^64-1。

创建表时，可以定义列的整数类型及其是否为无符号（UNSIGNED）。例如，创建一个存储员工薪水的表：

![](img/a4f337962ef1c0294751807526423018_9.png)

![](img/a4f337962ef1c0294751807526423018_11.png)

```sql
CREATE TABLE employees (
    salary INT(5) UNSIGNED
);
```

![](img/a4f337962ef1c0294751807526423018_13.png)

在上面的例子中，`INT(5)` 的显示宽度为5，`UNSIGNED` 表示该列只存储非负整数。对于无符号的 `INT` 类型，最大可存储值约为42亿（4,294,967,295）。如果使用有符号的 `INT`，范围则约为负21亿到正21亿（-2,147,483,648 到 2,147,483,647）。

![](img/a4f337962ef1c0294751807526423018_15.png)

![](img/a4f337962ef1c0294751807526423018_17.png)

## 显示宽度与零填充

接下来，我们探讨整数类型的显示宽度和零填充属性。显示宽度并不限制可存储的数值范围，而是影响某些客户端显示数字时的格式。

![](img/a4f337962ef1c0294751807526423018_19.png)

例如，定义 `INT(11)` 表示在显示时，该列通常会预留出11个字符的宽度。如果存储的数字位数少于显示宽度，默认情况下会以空格在左侧填充。

![](img/a4f337962ef1c0294751807526423018_20.png)

我们可以使用 `ZEROFILL` 属性，它自动隐含 `UNSIGNED` 属性。当数字位数小于显示宽度时，会在左侧用零填充。

![](img/a4f337962ef1c0294751807526423018_22.png)

以下是使用 `ZEROFILL` 的示例：

```sql
CREATE TABLE documents (
    doc_id INT(4) ZEROFILL
);
```

如果向 `doc_id` 列插入值 `11`，在查询显示时，它将会显示为 `0011`。

![](img/a4f337962ef1c0294751807526423018_24.png)

## 定点数类型

上一节我们介绍了整数，本节中我们来看看用于存储精确小数的定点数类型。定点数类型以字符串形式存储数字，非常适合需要精确计算的场景，如金融数据。

![](img/a4f337962ef1c0294751807526423018_26.png)

MySQL使用 `DECIMAL`（或同义词 `NUMERIC`）作为定点数类型。其语法为 `DECIMAL(M, D)`。

![](img/a4f337962ef1c0294751807526423018_28.png)

以下是 `DECIMAL` 类型参数的含义：

*   **M**：精度，表示总共可以存储的数字位数（整数位+小数位），最大值为65。
*   **D**：标度，表示小数点后可以存储的数字位数。

![](img/a4f337962ef1c0294751807526423018_30.png)

例如，创建一个存储汇率的表：

![](img/a4f337962ef1c0294751807526423018_32.png)

```sql
CREATE TABLE rates (
    rate DECIMAL(3, 2)
);
```

这里，`DECIMAL(3, 2)` 表示该列总共可以存储3位数字，其中小数点后占2位。因此，该列可以存储的有效值范围是 `-9.99` 到 `9.99`。如果插入 `4.65`，其中 `4` 是整数部分（属于精度的一部分），`.65` 是小数部分（由标度定义）。

![](img/a4f337962ef1c0294751807526423018_34.png)

如果不指定精度和标度，默认是 `DECIMAL(10, 0)`。MySQL允许的最大精度是65位数字。

![](img/a4f337962ef1c0294751807526423018_36.png)

本节课中我们一起学习了MySQL的数值数据类型。我们掌握了整数类型（TINYINT, SMALLINT, MEDIUMINT, INT, BIGINT）的存储范围和有符号/无符号区别，了解了显示宽度和 `ZEROFILL` 属性的作用，并重点学习了用于精确存储小数的 `DECIMAL(M, D)` 定点数类型及其精度和标度的概念。正确选择数值类型对于优化数据库存储和保证数据计算的精确性至关重要。
