在本节课中，我们将要学习 MongoDB 分片集群的基本概念。分片是一种将数据分布到多台服务器上的方法，用于处理大规模数据集和高吞吐量操作。

![](img/deb244284cff6b38cd1389b4a5672ba4_6.png)

![](img/deb244284cff6b38cd1389b4a5672ba4_7.png)

## 概述 📋

![](img/deb244284cff6b38cd1389b4a5672ba4_9.png)

![](img/deb244284cff6b38cd1389b4a5672ba4_11.png)

MongoDB 除了可以在单台服务器上运行，还可以工作在分片模式下。这意味着你可以使用多台服务器来平衡数据和负载，而不是仅仅依赖一台服务器。

![](img/deb244284cff6b38cd1389b4a5672ba4_13.png)

## 分片的作用 ⚖️

![](img/deb244284cff6b38cd1389b4a5672ba4_14.png)

![](img/deb244284cff6b38cd1389b4a5672ba4_16.png)

分片的主要目的是避免单台服务器过载。无论一台机器的性能多么强大，它都可能无法应对极高的查询速率，这可能会迅速耗尽 CPU 和内存资源。通过将负载分散到多台服务器上，我们可以：
*   减少单台机器的 RAM 消耗。
*   降低磁盘 I/O 的压力。
*   防止单台机器过载。

## 扩展类型 🔄

![](img/deb244284cff6b38cd1389b4a5672ba4_18.png)

![](img/deb244284cff6b38cd1389b4a5672ba4_20.png)

![](img/deb244284cff6b38cd1389b4a5672ba4_21.png)

扩展系统容量主要有两种方式：
*   **垂直扩展**：通过增加单个服务器的容量（如更快的 CPU、更大的内存）来提升性能。
*   **水平扩展**：这是我们本节课重点讨论的方式，它将负载分散到多台服务器上。你可以根据需要随时添加新的服务器来增加 MongoDB 集群的容量。

## 分片集群的组件 🧱

一个 MongoDB 分片集群主要由以下三个组件构成：

![](img/deb244284cff6b38cd1389b4a5672ba4_23.png)

![](img/deb244284cff6b38cd1389b4a5672ba4_24.png)

![](img/deb244284cff6b38cd1389b4a5672ba4_25.png)

以下是分片集群的核心组件列表：
1.  **分片**：每个分片包含集群数据的一个子集。分片本身可以是一个独立的 MongoDB 实例，但为了实现高可用性，通常被部署为一个**副本集**。
2.  **查询路由器**：它的作用是充当查询路由，为客户端应用程序和分片集群之间提供接口。客户端连接的是 `mongos` 实例，而不是直接连接分片。
3.  **配置服务器**：配置服务器存储集群的元数据和配置信息。自 MongoDB 3.4 版本起，配置服务器也必须部署为副本集，以确保配置信息的高可用性。

## 分片集群的优势 🚀

![](img/deb244284cff6b38cd1389b4a5672ba4_27.png)

采用分片集群架构能带来显著的好处：
*   **读写性能提升**：数据分布后，读写操作可以在多台服务器上并行进行，大大减轻了单台机器的压力。
*   **存储容量扩展**：存储容量不再受限于单块硬盘的大小，可以通过增加分片来线性扩展。
*   **高可用性**：即使一个或多个分片服务器不可用，集群仍然可以继续处理部分读写操作，系统会自动在其他可用分片间重新分配负载。

![](img/deb244284cff6b38cd1389b4a5672ba4_31.png)

## 核心概念：数据块 📦

![](img/deb244284cff6b38cd1389b4a5672ba4_32.png)

![](img/deb244284cff6b38cd1389b4a5672ba4_34.png)

上一节我们介绍了分片集群的宏观架构，本节中我们来看看数据是如何在分片间分布和管理的。这涉及到“数据块”的概念。

![](img/deb244284cff6b38cd1389b4a5672ba4_36.png)

![](img/deb244284cff6b38cd1389b4a5672ba4_38.png)

数据在分片间不是以单个文档为单位分布的，而是被组织成一个个的“块”。每个块包含基于分片键的某个范围内的数据。MongoDB 的均衡器会在后台自动将数据块从一个分片迁移到另一个分片，以保持集群中所有分片间的数据均匀分布。

![](img/deb244284cff6b38cd1389b4a5672ba4_40.png)

![](img/deb244284cff6b38cd1389b4a5672ba4_41.png)

这个机制是分片集群实现负载均衡的核心。我们将在后续课程中对数据块进行更深入的学习。

![](img/deb244284cff6b38cd1389b4a5672ba4_43.png)

![](img/deb244284cff6b38cd1389b4a5672ba4_44.png)

## 总结 🎯

![](img/deb244284cff6b38cd1389b4a5672ba4_46.png)

本节课我们一起学习了 MongoDB 分片集群的基础知识。我们了解到分片是一种水平扩展数据库的方式，通过将数据和负载分布到多台服务器上，可以有效提升系统的处理能力、存储容量和可用性。一个典型的分片集群由分片、查询路由器和配置服务器组成，数据以“块”为单位在分片间进行管理和迁移。理解这些基本概念是后续进行分片集群部署和管理的坚实基础。
