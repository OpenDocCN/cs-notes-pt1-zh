# 062：关节点与双连通分量 🔗

![](img/fb4998d581c01a1ea62c77f641d7ab39_1.png)

![](img/fb4998d581c01a1ea62c77f641d7ab39_3.png)

在本节课中，我们将学习图论中的两个重要概念：**关节点**和**双连通分量**。我们将了解什么是关节点，为什么它在网络设计中不受欢迎，以及如何通过算法找到并消除图中的关节点。

## 什么是关节点？🎯

![](img/fb4998d581c01a1ea62c77f641d7ab39_5.png)

在一个图中，如果移除某个顶点会导致图被分割成多个互不连通的组件，那么这个顶点就被称为**关节点**。

![](img/fb4998d581c01a1ea62c77f641d7ab39_7.png)

例如，在下图中，顶点3就是一个关节点。如果移除顶点3，图会被分割成两个部分：顶点1、4、2构成一个组件，顶点5、6构成另一个组件。

![](img/fb4998d581c01a1ea62c77f641d7ab39_8.png)

![](img/fb4998d581c01a1ea62c77f641d7ab39_1.png)

如果图代表一个网络（如道路网或计算机网络），我们通常不希望存在关节点。因为如果关节点出现问题，整个网络就会分裂成多个部分，影响连通性。

![](img/fb4998d581c01a1ea62c77f641d7ab39_7.png)

![](img/fb4998d581c01a1ea62c77f641d7ab39_10.png)

![](img/fb4998d581c01a1ea62c77f641d7ab39_11.png)

![](img/fb4998d581c01a1ea62c77f641d7ab39_13.png)

理想情况下，即使网络中某些部分发生故障，网络也应尽可能保持为一个完整的连通组件。因此，我们需要识别并处理关节点。

## 如何消除关节点？🔧

解决关节点问题的一个直接方法是：在由关节点连接的不同组件之间添加一条边。

以上图为例，如果在顶点4和顶点5之间添加一条边，那么即使移除顶点3，顶点5和6仍然可以通过顶点4与图的其余部分保持连通。这样，顶点3就不再是关节点了。

![](img/fb4998d581c01a1ea62c77f641d7ab39_10.png)

![](img/fb4998d581c01a1ea62c77f641d7ab39_15.png)

因此，消除关节点的核心思路是：**找到关节点，然后在它连接的不同组件之间添加额外的边**。

## 寻找关节点的算法 🧠

仅仅通过观察来识别关节点对于小图是可行的，但我们需要一个系统性的算法让计算机也能完成这个任务。以下是寻找关节点的算法步骤。

上一节我们介绍了关节点的概念和消除思路，本节中我们来看看如何通过算法自动找到它们。

### 第一步：生成深度优先搜索生成树

![](img/fb4998d581c01a1ea62c77f641d7ab39_17.png)

首先，为给定的图生成一棵**深度优先搜索生成树**。我们从一个起始顶点（例如顶点1）开始进行深度优先遍历，只记录遍历过程中首次访问边所构成的树形结构，忽略那些指向已访问顶点的边（回边）。

![](img/fb4998d581c01a1ea62c77f641d7ab39_19.png)

![](img/fb4998d581c01a1ea62c77f641d7ab39_21.png)

以下是从顶点1开始进行DFS可能得到的一棵生成树：
*   从1访问4。
*   从4访问3。
*   从3访问5。
*   从5访问6。
*   从6有一条边指向3（但3已被访问，这是一条回边，不计入树中）。
*   回溯到3，访问2。
*   从2有一条边指向1（回边，不计入树中）。

### 第二步：记录发现时间

![](img/fb4998d581c01a1ea62c77f641d7ab39_26.png)

![](img/fb4998d581c01a1ea62c77f641d7ab39_27.png)

为DFS生成树中的每个顶点标记一个**发现时间**，也称为DFS编号。这个编号就是顶点在深度优先遍历中被访问的顺序。

![](img/fb4998d581c01a1ea62c77f641d7ab39_29.png)

![](img/fb4998d581c01a1ea62c77f641d7ab39_30.png)

以下是各顶点的发现时间：
*   `d[1] = 1`
*   `d[4] = 2`
*   `d[3] = 3`
*   `d[5] = 4`
*   `d[6] = 5`
*   `d[2] = 6`

![](img/fb4998d581c01a1ea62c77f641d7ab39_32.png)

![](img/fb4998d581c01a1ea62c77f641d7ab39_17.png)

![](img/fb4998d581c01a1ea62c77f641d7ab39_34.png)

### 第三步：计算最低可到达编号

![](img/fb4998d581c01a1ea62c77f641d7ab39_36.png)

![](img/fb4998d581c01a1ea62c77f641d7ab39_37.png)

对于每个顶点 `v`，计算 `low[v]`。`low[v]` 表示从顶点 `v` 出发，通过**树边向下**走，然后最多使用**一条回边向上**，所能到达的顶点的**最小发现时间**。

以下是计算每个顶点 `low` 值的过程：
*   **low[1]**：从1出发，最低能回到自己，所以 `low[1] = 1`。
*   **low[2]**：从2出发，可以通过回边(2,1)直接到达1，所以 `low[2] = 1`。
*   **low[3]**：从3出发，可以走到2再通过回边到1，所以 `low[3] = 1`。
*   **low[4]**：从4出发，可以走到3->2->1，所以 `low[4] = 1`。
*   **low[5]**：从5出发，可以走到6，然后通过回边(6,3)到达3，所以 `low[5] = d[3] = 3`。
*   **low[6]**：从6出发，可以通过回边(6,3)直接到达3，所以 `low[6] = 3`。

![](img/fb4998d581c01a1ea62c77f641d7ab39_39.png)

![](img/fb4998d581c01a1ea62c77f641d7ab39_40.png)

### 第四步：应用关节点判定条件

现在，我们可以利用发现时间 `d` 和最低可到达编号 `low` 来判断关节点。判定规则针对DFS生成树中的**非根节点**：

![](img/fb4998d581c01a1ea62c77f641d7ab39_42.png)

对于树中的一条边 `(u, v)`，其中 `u` 是 `v` 的父节点。如果满足以下条件，则 `u` 是一个关节点：

**`low[v] >= d[u]`**

![](img/fb4998d581c01a1ea62c77f641d7ab39_44.png)

这个条件的含义是：顶点 `v` 及其后代**无法通过一条回边**绕过 `u` 到达 `u` 的祖先。因此，如果移除 `u`，`v` 所在的子树就会与图的其余部分断开。

让我们应用这个规则检查图中的顶点：
*   检查边 `(4, 3)`：`low[3] = 1`，`d[4] = 2`。`1 >= 2` 为假，所以4不是关节点。
*   检查边 `(3, 5)`：`low[5] = 3`，`d[3] = 3`。`3 >= 3` 为真，所以**顶点3是关节点**。

![](img/fb4998d581c01a1ea62c77f641d7ab39_46.png)

![](img/fb4998d581c01a1ea62c77f641d7ab39_47.png)

这与我们之前直观观察的结果一致。

### 第五步：根节点的特殊处理

上述规则不适用于DFS树的根节点。判断根节点是否为关节点有一个更简单的条件：**如果根节点在DFS树中有两个或更多的子节点，那么它就是关节点**。因为移除根节点会使其各个子树彼此分离。

## 算法总结与回顾 📝

![](img/fb4998d581c01a1ea62c77f641d7ab39_52.png)

本节课中我们一起学习了关节点与双连通分量的核心知识。

以下是寻找并消除关节点的完整流程总结：
1.  **生成DFS树**：对图进行深度优先搜索，得到一棵生成树。
2.  **标记发现时间**：记录每个顶点在DFS中被访问的顺序 `d[u]`。
3.  **计算Low值**：计算每个顶点通过树边和最多一条回边能到达的最小发现时间 `low[u]`。
4.  **判定关节点**：
    *   对于**非根节点** `u`，如果存在一个子节点 `v` 满足 `low[v] >= d[u]`，则 `u` 是关节点。
    *   对于**根节点**，如果它拥有两个或更多子节点，则它是关节点。
5.  **消除关节点**：一旦找到关节点，可以在由该关节点连接的不同连通分量之间添加边，从而使图变为双连通图（没有关节点的图）。

通过这个算法，我们就能系统地分析任何图的脆弱点，并通过增加冗余连接来增强网络的鲁棒性。
