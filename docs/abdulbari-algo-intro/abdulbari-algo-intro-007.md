## 循环变量倍增的循环分析 🚀

![](img/a73ec6efc7ff731008a24a30f856f933_12.png)

![](img/a73ec6efc7ff731008a24a30f856f933_14.png)

上一节我们介绍了线性递增循环的时间复杂度。本节中，我们来看看一个不同的循环结构。在这个代码片段中，循环变量 `i` 从1开始，每次迭代后乘以2，直到 `i` 大于等于 `n`。

![](img/a73ec6efc7ff731008a24a30f856f933_16.png)

![](img/a73ec6efc7ff731008a24a30f856f933_17.png)

![](img/a73ec6efc7ff731008a24a30f856f933_19.png)

![](img/a73ec6efc7ff731008a24a30f856f933_20.png)

```c
for (i = 1; i < n; i = i * 2) {
    // 循环体语句
}
```

![](img/a73ec6efc7ff731008a24a30f856f933_22.png)

我们不能直接说这个循环会执行 `n` 次，因为 `i` 的增长是指数级的。我们需要分析它实际执行的次数。

![](img/a73ec6efc7ff731008a24a30f856f933_24.png)

![](img/a73ec6efc7ff731008a24a30f856f933_25.png)

![](img/a73ec6efc7ff731008a24a30f856f933_26.png)

![](img/a73ec6efc7ff731008a24a30f856f933_28.png)

以下是循环变量 `i` 的变化过程：
*   第一次迭代后：`i = 1 * 2 = 2`
*   第二次迭代后：`i = 2 * 2 = 4`
*   第三次迭代后：`i = 4 * 2 = 8`
*   第 `k` 次迭代后：`i = 2^k`

![](img/a73ec6efc7ff731008a24a30f856f933_30.png)

![](img/a73ec6efc7ff731008a24a30f856f933_31.png)

![](img/a73ec6efc7ff731008a24a30f856f933_33.png)

循环终止的条件是 `i >= n`。我们假设在第 `k` 次迭代后，`i` 首次大于等于 `n`。因此，我们可以建立方程：
`2^k >= n`

![](img/a73ec6efc7ff731008a24a30f856f933_35.png)

为了求解 `k`，我们取对数：
`k = log₂(n)`

![](img/a73ec6efc7ff731008a24a30f856f933_37.png)

![](img/a73ec6efc7ff731008a24a30f856f933_39.png)

![](img/a73ec6efc7ff731008a24a30f856f933_41.png)

因此，循环体语句将执行 **O(log₂ n)** 次。这表明，当循环变量以固定倍数（如2、3等）增长时，时间复杂度是对数级的。

![](img/a73ec6efc7ff731008a24a30f856f933_43.png)

![](img/a73ec6efc7ff731008a24a30f856f933_44.png)

## 与线性递增循环的对比 📈

![](img/a73ec6efc7ff731008a24a30f856f933_46.png)

![](img/a73ec6efc7ff731008a24a30f856f933_47.png)

![](img/a73ec6efc7ff731008a24a30f856f933_49.png)

为了加深理解，我们可以将倍增循环与标准的线性递增循环进行对比。

![](img/a73ec6efc7ff731008a24a30f856f933_51.png)

![](img/a73ec6efc7ff731008a24a30f856f933_52.png)

![](img/a73ec6efc7ff731008a24a30f856f933_53.png)

![](img/a73ec6efc7ff731008a24a30f856f933_55.png)

对于线性递增循环：
```c
for (i = 1; i <= n; i++) {
    // 循环体语句
}
```
循环变量 `i` 从1增加到 `n`，每次增加1。执行次数 `k` 等于 `n`，因此时间复杂度是 **O(n)**。

![](img/a73ec6efc7ff731008a24a30f856f933_56.png)

![](img/a73ec6efc7ff731008a24a30f856f933_58.png)

![](img/a73ec6efc7ff731008a24a30f856f933_60.png)

![](img/a73ec6efc7ff731008a24a30f856f933_62.png)

![](img/a73ec6efc7ff731008a24a30f856f933_64.png)

对于倍增循环：
循环变量 `i` 从1开始，每次乘以2。经过 `k` 次乘法后，`i` 的值变为 `2^k`。当 `2^k >= n` 时循环停止，因此 `k = log₂ n`，时间复杂度是 **O(log₂ n)**。

![](img/a73ec6efc7ff731008a24a30f856f933_66.png)

![](img/a73ec6efc7ff731008a24a30f856f933_67.png)

这个对比清晰地展示了循环变量的更新方式如何从根本上影响算法的执行次数。

![](img/a73ec6efc7ff731008a24a30f856f933_69.png)

![](img/a73ec6efc7ff731008a24a30f856f933_70.png)

![](img/a73ec6efc7ff731008a24a30f856f933_71.png)

## 对数时间的取整问题 🔢

![](img/a73ec6efc7ff731008a24a30f856f933_77.png)

![](img/a73ec6efc7ff731008a24a30f856f933_78.png)

由于 `log₂ n` 可能不是整数（例如 `log₂ 10 ≈ 3.32`），我们需要确定在实际计算循环次数时，应该向上取整还是向下取整。

![](img/a73ec6efc7ff731008a24a30f856f933_80.png)

![](img/a73ec6efc7ff731008a24a30f856f933_81.png)

让我们通过两个例子来验证：
*   当 `n = 8` 时，`i` 的变化为：1, 2, 4, 8。循环执行了3次。`log₂ 8 = 3`，恰好是整数。
*   当 `n = 10` 时，`i` 的变化为：1, 2, 4, 8, 16。循环执行了4次。`log₂ 10 ≈ 3.32`，而实际执行次数是4。

从例子可以看出，为了保证循环能正确终止，我们需要对 `log₂ n` 的结果进行**向上取整 (ceil)**。在时间复杂度的大O表示法中，我们通常直接写作 **O(log n)**，隐含了取整操作。

## 循环变量倍减的循环分析 📉

![](img/a73ec6efc7ff731008a24a30f856f933_83.png)

![](img/a73ec6efc7ff731008a24a30f856f933_84.png)

![](img/a73ec6efc7ff731008a24a30f856f933_86.png)

与倍增循环类似，如果循环变量每次迭代被除以一个常数，时间复杂度同样是对数级的。

![](img/a73ec6efc7ff731008a24a30f856f933_88.png)

![](img/a73ec6efc7ff731008a24a30f856f933_89.png)

![](img/a73ec6efc7ff731008a24a30f856f933_91.png)

![](img/a73ec6efc7ff731008a24a30f856f933_92.png)

![](img/a73ec6efc7ff731008a24a30f856f933_94.png)

考虑以下代码：
```c
for (i = n; i >= 1; i = i / 2) {
    // 循环体语句
}
```

![](img/a73ec6efc7ff731008a24a30f856f933_96.png)

![](img/a73ec6efc7ff731008a24a30f856f933_98.png)

![](img/a73ec6efc7ff731008a24a30f856f933_99.png)

以下是循环变量 `i` 的变化过程：
*   第一次迭代后：`i = n / 2`
*   第二次迭代后：`i = n / 4`
*   第 `k` 次迭代后：`i = n / (2^k)`

![](img/a73ec6efc7ff731008a24a30f856f933_101.png)

![](img/a73ec6efc7ff731008a24a30f856f933_103.png)

![](img/a73ec6efc7ff731008a24a30f856f933_105.png)

![](img/a73ec6efc7ff731008a24a30f856f933_106.png)

循环终止的条件是 `i < 1`。假设在第 `k` 次迭代后，`i` 首次小于1。我们可以建立方程：
`n / (2^k) < 1` => `2^k > n` => `k > log₂ n`

![](img/a73ec6efc7ff731008a24a30f856f933_108.png)

![](img/a73ec6efc7ff731008a24a30f856f933_109.png)

![](img/a73ec6efc7ff731008a24a30f856f933_110.png)

因此，这个循环的时间复杂度也是 **O(log₂ n)**。无论是倍增还是倍减，只要变化是指数级的，时间复杂度就是对数级。

![](img/a73ec6efc7ff731008a24a30f856f933_112.png)

![](img/a73ec6efc7ff731008a24a30f856f933_113.png)

![](img/a73ec6efc7ff731008a24a30f856f933_114.png)

现在，让我们快速分析几种其他常见的循环模式，以巩固理解。

![](img/a73ec6efc7ff731008a24a30f856f933_121.png)

![](img/a73ec6efc7ff731008a24a30f856f933_122.png)

**示例1：循环条件为 `i * i <= n`**
```c
for (i = 0; i * i <= n; i++) {
    // 循环体语句
}
```
循环终止条件是 `i² > n`，因此 `i` 大约增长到 `√n`。时间复杂度为 **O(√n)**。

![](img/a73ec6efc7ff731008a24a30f856f933_126.png)

![](img/a73ec6efc7ff731008a24a30f856f933_127.png)

![](img/a73ec6efc7ff731008a24a30f856f933_128.png)

![](img/a73ec6efc7ff731008a24a30f856f933_130.png)

**示例2：多个独立顺序循环**
```c
for (i = 0; i < n; i++) { // 循环A
    // 语句
}
for (j = 0; j < n; j++) { // 循环B
    // 语句
}
```
循环A和循环B是顺序执行的，不是嵌套。总时间复杂度为 O(n) + O(n) = **O(n)**。

![](img/a73ec6efc7ff731008a24a30f856f933_132.png)

![](img/a73ec6efc7ff731008a24a30f856f933_134.png)

![](img/a73ec6efc7ff731008a24a30f856f933_136.png)

![](img/a73ec6efc7ff731008a24a30f856f933_138.png)

**示例3：嵌套循环（外层线性，内层对数）**
```c
for (i = 0; i < n; i++) { // 外层：O(n)
    for (j = 1; j < n; j = j * 2) { // 内层：O(log n)
        // 语句
    }
}
```
外层循环执行 `n` 次，每次内层循环执行 `log n` 次。总时间复杂度为 **O(n log n)**。

根据以上分析，我们可以总结出一些常见循环模式的时间复杂度公式：

![](img/a73ec6efc7ff731008a24a30f856f933_155.png)

![](img/a73ec6efc7ff731008a24a30f856f933_156.png)

![](img/a73ec6efc7ff731008a24a30f856f933_158.png)

![](img/a73ec6efc7ff731008a24a30f856f933_160.png)

![](img/a73ec6efc7ff731008a24a30f856f933_162.png)

以下是基于循环更新方式的复杂度速查：
*   **线性更新**（递增或递减）：`for(i=0; i<n; i++)` 或 `for(i=n; i>0; i--)` -> 时间复杂度为 **O(n)**。
    *   即使步长不是1（如 `i+=2` 或 `i-=5`），复杂度仍然是 **O(n)**，因为常数倍不影响渐近趋势。
*   **指数倍增更新**：`for(i=1; i<n; i=i*2)` -> 时间复杂度为 **O(log n)**。
    *   如果乘以常数 `c`（如 `i=i*3`），复杂度为 **O(log_c n)**，在大O记法中仍为 **O(log n)**。
*   **指数倍减更新**：`for(i=n; i>1; i=i/2)` -> 时间复杂度为 **O(log n)**。
*   **平方根边界**：`for(i=0; i*i<n; i++)` -> 时间复杂度为 **O(√n)**。
*   **嵌套循环**：需要将各层循环的复杂度相乘。
    *   例如，外层O(n)，内层O(log n)，则总复杂度为 **O(n log n)**。

本节课中我们一起学习了如何分析非标准循环的时间复杂度。核心在于识别循环变量的更新模式：线性更新导致线性时间 O(n)，而几何级数（乘以或除以常数）的更新则导致对数时间 O(log n)。我们还学习了如何处理对数结果的取整问题，并总结了多种常见循环模式的复杂度结论。掌握这些模式是进行算法效率分析的基础。
