### 简单循环分析

![](img/d9811765b60d3abd823316ab10d1e55f_5.png)

首先，我们来看一个简单的 `for` 循环。我们只关注循环体内的语句执行了多少次。

![](img/d9811765b60d3abd823316ab10d1e55f_7.png)

```c
for (i=0; i<n; i++) {
    // 某个语句
}
```

![](img/d9811765b60d3abd823316ab10d1e55f_9.png)

我们知道，循环条件 `i<n` 会检查 `n+1` 次，循环体语句会执行 `n` 次。在分析时间复杂度时，我们主要关注循环体语句的执行次数，因为它决定了多项式的最高次项。因此，这个循环的时间复杂度是 **O(n)**。

![](img/d9811765b60d3abd823316ab10d1e55f_11.png)

![](img/d9811765b60d3abd823316ab10d1e55f_12.png)

### 递减循环

![](img/d9811765b60d3abd823316ab10d1e55f_18.png)

上一节我们分析了递增循环，本节中我们来看看递减循环。循环的方向（递增或递减）不影响其执行次数。

![](img/d9811765b60d3abd823316ab10d1e55f_20.png)

![](img/d9811765b60d3abd823316ab10d1e55f_22.png)

```c
for (i=n; i>0; i--) {
    // 某个语句
}
```

![](img/d9811765b60d3abd823316ab10d1e55f_24.png)

无论是从 `1` 数到 `n`，还是从 `n` 数到 `1`，都需要 `n` 步。因此，这个循环的时间复杂度同样是 **O(n)**。

![](img/d9811765b60d3abd823316ab10d1e55f_26.png)

![](img/d9811765b60d3abd823316ab10d1e55f_28.png)

### 步长大于1的循环

![](img/d9811765b60d3abd823316ab10d1e55f_30.png)

接下来，我们分析步长不为1的循环。这会影响循环体执行的次数。

![](img/d9811765b60d3abd823316ab10d1e55f_32.png)

```c
for (i=1; i<n; i=i+2) {
    // 某个语句
}
```

![](img/d9811765b60d3abd823316ab10d1e55f_34.png)

![](img/d9811765b60d3abd823316ab10d1e55f_35.png)

在这个循环中，变量 `i` 每次增加 `2`。因此，循环体执行的次数大约是 `n/2` 次。即使步长是其他常数（例如 `i=i+5`），执行次数也只是 `n/5` 次。在计算大O复杂度时，常数因子会被忽略。所以，无论步长是多少，只要它是常数，时间复杂度就仍然是 **O(n)**。

![](img/d9811765b60d3abd823316ab10d1e55f_37.png)

### 嵌套循环

![](img/d9811765b60d3abd823316ab10d1e55f_42.png)

![](img/d9811765b60d3abd823316ab10d1e55f_44.png)

现在，我们来看一个经典的嵌套循环结构。

![](img/d9811765b60d3abd823316ab10d1e55f_46.png)

![](img/d9811765b60d3abd823316ab10d1e55f_47.png)

```c
for (i=0; i<n; i++) {
    for (j=0; j<n; j++) {
        // 某个语句
    }
}
```

![](img/d9811765b60d3abd823316ab10d1e55f_49.png)

![](img/d9811765b60d3abd823316ab10d1e55f_50.png)

外层循环执行 `n` 次。对于外层循环的每一次迭代，内层循环也执行 `n` 次。因此，内层循环的语句总共执行了 `n * n = n²` 次。所以，这个嵌套循环的时间复杂度是 **O(n²)**。

![](img/d9811765b60d3abd823316ab10d1e55f_52.png)

![](img/d9811765b60d3abd823316ab10d1e55f_54.png)

### 内层循环次数变化的嵌套循环

![](img/d9811765b60d3abd823316ab10d1e55f_56.png)

![](img/d9811765b60d3abd823316ab10d1e55f_57.png)

![](img/d9811765b60d3abd823316ab10d1e55f_59.png)

上一个例子中内层循环次数是固定的。现在我们分析一个内层循环次数会随外层循环变化的例子。

```c
for (i=0; i<n; i++) {
    for (j=0; j<i; j++) {
        // 某个语句
    }
}
```

为了分析它，我们可以通过追踪变量值来计算语句执行的总次数。

以下是执行过程的追踪：
*   当 `i = 0` 时，内层循环条件 `j<0` 不成立，语句执行 **0** 次。
*   当 `i = 1` 时，内层循环 `j` 取值 `0`，语句执行 **1** 次。
*   当 `i = 2` 时，内层循环 `j` 取值 `0, 1`，语句执行 **2** 次。
*   当 `i = 3` 时，内层循环 `j` 取值 `0, 1, 2`，语句执行 **3** 次。
*   ...
*   当 `i = n-1` 时，语句执行 **n-1** 次。

因此，语句执行的总次数是 `0 + 1 + 2 + 3 + ... + (n-1)`。

![](img/d9811765b60d3abd823316ab10d1e55f_61.png)

![](img/d9811765b60d3abd823316ab10d1e55f_63.png)

这是一个等差数列求和，其和为：
**公式： (n-1)*n / 2 = (n² - n) / 2**

这个多项式的最高次项是 `n²`。所以，尽管这个循环的执行模式与上一个不同，但其时间复杂度仍然是 **O(n²)**。

![](img/d9811765b60d3abd823316ab10d1e55f_65.png)

![](img/d9811765b60d3abd823316ab10d1e55f_66.png)

### 条件依赖于累加和的循环

![](img/d9811765b60d3abd823316ab10d1e55f_68.png)

![](img/d9811765b60d3abd823316ab10d1e55f_69.png)

最后，我们分析一个更复杂的循环，其终止条件不是简单的计数器。

```c
p = 0;
for (i=1; p <= n; i++) {
    p = p + i;
}
```

这个循环的终止条件是 `p <= n`，而 `p` 是每次迭代中 `i` 的累加和。我们无法直接看出它执行了多少次。

我们假设循环执行了 `k` 次后停止。那么：
*   第一次迭代后，`p = 1`
*   第二次迭代后，`p = 1 + 2`
*   第三次迭代后，`p = 1 + 2 + 3`
*   ...
*   第 `k` 次迭代后，`p = 1 + 2 + 3 + ... + k`

![](img/d9811765b60d3abd823316ab10d1e55f_71.png)

![](img/d9811765b60d3abd823316ab10d1e55f_72.png)

`p` 的值是前 `k` 个自然数的和：
**公式： p = k*(k+1) / 2**

![](img/d9811765b60d3abd823316ab10d1e55f_74.png)

![](img/d9811765b60d3abd823316ab10d1e55f_75.png)

![](img/d9811765b60d3abd823316ab10d1e55f_76.png)

循环会在 `p > n` 时停止。因此，我们可以建立不等式：
**公式： k*(k+1) / 2 > n**

![](img/d9811765b60d3abd823316ab10d1e55f_78.png)

![](img/d9811765b60d3abd823316ab10d1e55f_79.png)

解这个不等式，`k²` 是主导项，所以 `k` 的数量级大约是 `√n`。因此，这个循环的时间复杂度是 **O(√n)**。

![](img/d9811765b60d3abd823316ab10d1e55f_81.png)

![](img/d9811765b60d3abd823316ab10d1e55f_83.png)

### 总结

![](img/d9811765b60d3abd823316ab10d1e55f_85.png)

![](img/d9811765b60d3abd823316ab10d1e55f_86.png)

本节课中我们一起学习了如何分析几种常见循环结构的时间复杂度：
1.  简单的单层循环，时间复杂度为 **O(n)**。
2.  步长为常数的循环，时间复杂度仍为 **O(n)**。
3.  两层循环次数固定的嵌套循环，时间复杂度为 **O(n²)**。
4.  内层循环次数依赖于外层的嵌套循环，通过求和公式分析，时间复杂度也为 **O(n²)**。
5.  终止条件依赖于累加值的循环，需要通过建立方程求解执行次数，本例中得到的时间复杂度为 **O(√n)**。

![](img/d9811765b60d3abd823316ab10d1e55f_88.png)

核心方法是计算关键语句的执行次数，并找出其关于输入规模 `n` 的函数表达式中增长最快的项（最高次项）。在下一个视频中，我们将看到更多不同类型的代码分析示例。
