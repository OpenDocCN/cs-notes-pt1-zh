# 050：矩阵链乘法问题与动态规划解法 🧮

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_1.png)

在本节课中，我们将要学习矩阵链乘法问题。这是一个经典的动态规划问题，其核心目标是找到一种最优的矩阵相乘顺序，使得完成整个链式乘法所需的总标量乘法次数最少。我们将从理解矩阵乘法的基础开始，逐步推导出动态规划的递推公式，并通过一个完整的例子演示如何填充表格来找到最优解。

## 矩阵乘法基础

首先，我们需要理解两个矩阵相乘的条件和成本。

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_7.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_8.png)

两个矩阵A和B可以相乘的条件是：**第一个矩阵A的列数必须等于第二个矩阵B的行数**。如果A的维度是 `p × q`，B的维度是 `q × r`，那么它们可以相乘，结果矩阵C的维度将是 `p × r`。

计算乘积矩阵C中每个元素 `C[i][j]` 的公式是：
```
C[i][j] = Σ (A[i][k] * B[k][j])， 其中 k 从 1 到 q
```
这意味着计算C的每个元素需要进行 `q` 次乘法。由于C有 `p * r` 个元素，因此两个矩阵相乘的总乘法次数为：
```
总乘法次数 = p × q × r
```
这个数值也被称为这次矩阵乘法的“成本”。

## 什么是矩阵链乘法问题？

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_10.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_11.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_12.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_14.png)

现在，假设我们有一个矩阵链 `A1, A2, A3, ..., An`，并且我们知道每个矩阵的维度（例如A1是 `d0 × d1`，A2是 `d1 × d2`，以此类推）。由于相邻矩阵的维度匹配，整个链理论上可以相乘。

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_16.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_18.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_19.png)

然而，矩阵乘法满足结合律，我们可以通过添加括号来改变相乘的顺序。例如，对于四个矩阵 `A1 A2 A3 A4`，可能的相乘顺序有：
1. `((A1 A2) A3) A4`
2. `(A1 (A2 A3)) A4`
3. `(A1 A2) (A3 A4)`
4. `A1 ((A2 A3) A4)`
5. `A1 (A2 (A3 A4))`

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_21.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_23.png)

**矩阵链乘法问题**就是：从所有可能的括号化方案中，找出一种使得完成整个链乘法所需总成本（即总标量乘法次数）最小的方案。

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_25.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_26.png)

对于 `n` 个矩阵，可能的括号化方案总数是一个卡特兰数 `C(n-1)`。当 `n` 较大时，暴力枚举所有方案是不可行的，因此我们需要更高效的算法——动态规划。

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_28.png)

## 动态规划解法

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_29.png)

动态规划通过将大问题分解为重叠子问题，并存储子问题的解来避免重复计算，从而高效地解决问题。对于矩阵链乘法，我们采用自底向上的填表法。

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_31.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_33.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_35.png)

### 定义子问题与表格

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_37.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_39.png)

我们定义子问题：`M[i][j]` 表示计算矩阵链 `Ai ... Aj`（即从第 `i` 个到第 `j` 个矩阵）所需的最小标量乘法次数。

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_41.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_43.png)

我们还需要一个辅助表格 `S[i][j]`，用于记录在得到 `M[i][j]` 这个最优解时，我们在哪个位置 `k` 进行了分割（即括号的位置），使得 `(Ai ... Ak)` 和 `(A(k+1) ... Aj)` 先分别计算，然后再相乘。

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_45.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_47.png)

假设矩阵链的维度信息存储在一个数组 `d` 中，其中矩阵 `Ai` 的维度为 `d[i-1] × d[i]`。对于 `n` 个矩阵，数组 `d` 的长度为 `n+1`。

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_49.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_50.png)

### 递推公式

对于子问题 `M[i][j]`，我们需要考虑所有可能的分割点 `k`（`i ≤ k < j`）。如果我们选择在 `k` 处分割，那么：
1.  先计算左边子链 `Ai ... Ak` 的最小成本：`M[i][k]`
2.  再计算右边子链 `A(k+1) ... Aj` 的最小成本：`M[k+1][j]`
3.  最后将这两个结果矩阵相乘的成本：`d[i-1] * d[k] * d[j]`（左边结果矩阵维度为 `d[i-1] × d[k]`，右边为 `d[k] × d[j]`）

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_52.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_54.png)

因此，在 `k` 处分割的总成本是这三部分之和。我们需要遍历所有可能的 `k`，找到使总成本最小的那个。递推公式如下：

```
如果 i == j，则 M[i][j] = 0 （单个矩阵无需相乘）
否则：
M[i][j] = min { M[i][k] + M[k+1][j] + d[i-1]*d[k]*d[j] }， 其中 i ≤ k < j
S[i][j] = 使上述表达式取最小值的 k
```

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_56.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_58.png)

### 填表示例

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_60.png)

让我们通过一个具体例子来演示填表过程。假设有4个矩阵：
- A1: 5 × 4
- A2: 4 × 6
- A3: 6 × 2
- A4: 2 × 7

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_62.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_64.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_66.png)

那么维度数组 `d = [5, 4, 6, 2, 7]`。

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_68.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_70.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_71.png)

我们初始化表格 `M` 和 `S`。首先，对角线 `M[i][i] = 0`，因为单个矩阵没有乘法成本。

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_72.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_74.png)

**第一步：计算长度为2的链（即相邻矩阵对）**
- `M[1][2]`：计算 A1 * A2。成本 = 5 * 4 * 6 = 120。`S[1][2] = 1`（在第一个矩阵后分割，但长度为2时只有一种可能）。
- `M[2][3]`：计算 A2 * A3。成本 = 4 * 6 * 2 = 48。`S[2][3] = 2`。
- `M[3][4]`：计算 A3 * A4。成本 = 6 * 2 * 7 = 84。`S[3][4] = 3`。

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_76.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_77.png)

**第二步：计算长度为3的链**
- `M[1][3]`：计算 A1 A2 A3。有两种括号化方式：
    1. `(A1 A2) A3`：成本 = `M[1][1] + M[2][3] + 5*4*2 = 0 + 48 + 40 = 88`
    2. `A1 (A2 A3)`：成本 = `M[1][2] + M[3][3] + 5*6*2 = 120 + 0 + 60 = 180`
    取最小值 88，所以 `M[1][3] = 88`，`S[1][3] = 1`（在k=1处分割，即 `(A1)(A2 A3)` 这种形式，但结合顺序是 `(A1 (A2 A3))`？注意：`S[1][3]=1` 表示分割为 `(A1)` 和 `(A2 A3)`，然后组合成 `(A1 (A2 A3))`）。
- `M[2][4]`：计算 A2 A3 A4。有两种方式：
    1. `(A2 A3) A4`：成本 = `M[2][2] + M[3][4] + 4*6*7 = 0 + 84 + 168 = 252`
    2. `A2 (A3 A4)`：成本 = `M[2][3] + M[4][4] + 4*2*7 = 48 + 0 + 56 = 104`
    取最小值 104，所以 `M[2][4] = 104`，`S[2][4] = 3`（在k=3处分割，即 `(A2 A3)(A4)`，组合成 `((A2 A3) A4)`？这里 `S[2][4]=3` 表示分割为 `(A2 A3)` 和 `(A4)`，即 `((A2 A3) A4)`）。

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_79.png)

**第三步：计算长度为4的整个链**
- `M[1][4]`：计算 A1 A2 A3 A4。有三种分割方式：
    1. k=1: `(A1)(A2 A3 A4)` -> 成本 = `M[1][1] + M[2][4] + 5*4*7 = 0 + 104 + 140 = 244`
    2. k=2: `(A1 A2)(A3 A4)` -> 成本 = `M[1][2] + M[3][4] + 5*6*7 = 120 + 84 + 210 = 414`
    3. k=3: `(A1 A2 A3)(A4)` -> 成本 = `M[1][3] + M[4][4] + 5*2*7 = 88 + 0 + 70 = 158`
    取最小值 158，所以 `M[1][4] = 158`，`S[1][4] = 3`。

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_81.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_83.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_84.png)

因此，最小乘法总成本为158。根据 `S` 表可以重构最优括号化方案：
- `S[1][4] = 3`，表示在3处分割：`(A1 A2 A3) (A4)`。
- 再看 `S[1][3] = 1`，表示 `(A1 A2 A3)` 这个子链在1处分割：`(A1) (A2 A3)`。
- 而 `S[2][3] = 2`，表示 `(A2 A3)` 就是 `(A2)(A3)`。

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_86.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_87.png)

所以最终的最优计算顺序是：`(A1 (A2 A3)) A4`。即先计算 `A2 * A3`（成本48），再计算其结果与 `A1` 相乘（成本 `5*6*2=60`，加上子成本总88），最后再与 `A4` 相乘（成本 `5*2*7=70`，总成本88+70=158）。

### 时间复杂度分析

我们需要填充 `M` 和 `S` 表格的上三角部分（因为 `i ≤ j`）。对于 `n` 个矩阵，表格大约有 `n²/2` 个元素需要计算。

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_89.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_90.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_92.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_94.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_96.png)

在计算每个 `M[i][j]` 时，我们需要遍历 `i` 到 `j-1` 的所有 `k` 值，在最坏情况下（即计算 `M[1][n]` 时）需要 `O(n)` 次比较。

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_98.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_99.png)

![](img/5f79e634b4fc3da4b20cd9dae262e4f9_100.png)

因此，算法的总时间复杂度为 `O(n³)`。虽然仍然是多项式时间，但比暴力枚举所有卡特兰数种方案要高效得多。
