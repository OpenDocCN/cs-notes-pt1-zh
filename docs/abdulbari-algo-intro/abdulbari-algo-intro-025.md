# 025：分治递推关系求解 - T(n)=T(n/2)+n 🔍

![](img/e4521b23b2c3683a231dc65a04da5b27_1.png)

![](img/e4521b23b2c3683a231dc65a04da5b27_2.png)

![](img/e4521b23b2c3683a231dc65a04da5b27_3.png)

在本节课中，我们将学习如何使用两种方法求解分治类型的递推关系：**T(n) = T(n/2) + n**。我们将详细讲解**递归树法**和**代入法**，并通过几何图形直观理解求和过程。最终，我们将得出该递推关系的时间复杂度为 **O(n)**。

![](img/e4521b23b2c3683a231dc65a04da5b27_5.png)

## 递归树法求解 🧮

上一节我们介绍了递推关系的基本概念，本节中我们来看看如何使用递归树法进行求解。递归树法通过将递归调用过程可视化为一棵树，并计算每一层的工作量来求解总时间。

首先，我们写出递推关系：**T(n) = T(n/2) + n**。等号右边的 **n** 代表当前步骤花费的时间，**T(n/2)** 代表递归调用子问题所花费的时间。

![](img/e4521b23b2c3683a231dc65a04da5b27_7.png)

![](img/e4521b23b2c3683a231dc65a04da5b27_8.png)

以下是构建递归树的步骤：
1.  根节点代表 **T(n)**，其成本为 **n**。
2.  根节点产生一个子节点，代表 **T(n/2)**，其成本为 **n/2**。
3.  子节点 **T(n/2)** 继续产生一个子节点 **T(n/4)**，成本为 **n/4**。
4.  此过程持续进行，直到问题规模缩小到 **1**，即 **T(1)**。

假设递归进行 **k** 次后问题规模变为 **1**，即 **n / 2^k = 1**。由此可得 **k = log₂ n**。

![](img/e4521b23b2c3683a231dc65a04da5b27_10.png)

现在，我们将递归树每一层的成本相加：
*   第 1 层（根节点）成本：**n**
*   第 2 层成本：**n/2**
*   第 3 层成本：**n/4**
*   ...
*   第 k 层成本：**n / 2^(k-1)**

![](img/e4521b23b2c3683a231dc65a04da5b27_12.png)

![](img/e4521b23b2c3683a231dc65a04da5b27_14.png)

总成本 **T(n)** 是所有这些成本之和：
**T(n) = n + n/2 + n/4 + ... + n / 2^(k-1)**

![](img/e4521b23b2c3683a231dc65a04da5b27_16.png)

![](img/e4521b23b2c3683a231dc65a04da5b27_18.png)

![](img/e4521b23b2c3683a231dc65a04da5b27_20.png)

我们可以提取公因子 **n**：
**T(n) = n * (1 + 1/2 + 1/4 + ... + 1/2^(k-1))**

![](img/e4521b23b2c3683a231dc65a04da5b27_22.png)

括号内是一个等比数列求和。当项数趋近于无穷时，该和收敛于 **2**。在我们的有限项求和中，它小于 **2**。因此，我们可以得出：
**T(n) ≈ n * 2 = O(n)**

![](img/e4521b23b2c3683a231dc65a04da5b27_24.png)

![](img/e4521b23b2c3683a231dc65a04da5b27_25.png)

为了更直观地理解这个求和，可以想象一个完整的圆代表数字 **1**。将其一半涂色（1/2），再将剩下部分的一半涂色（1/4），如此继续。即使无限进行下去，所有涂色部分的总和也永远不会超过整个圆（1）。这说明了该级数的和有界性。

![](img/e4521b23b2c3683a231dc65a04da5b27_27.png)

## 代入法求解 🔁

![](img/e4521b23b2c3683a231dc65a04da5b27_29.png)

![](img/e4521b23b2c3683a231dc65a04da5b27_30.png)

上一节我们使用递归树法得到了结果，本节中我们使用代入法进行验证。代入法通过反复将递推式右侧的 **T()** 项用其定义进行展开来求解。

![](img/e4521b23b2c3683a231dc65a04da5b27_32.png)

我们从原始递推式开始：
**T(n) = T(n/2) + n**

将等式右边的 **T(n/2)** 用其自身定义展开：
**T(n/2) = T(n/4) + n/2**

![](img/e4521b23b2c3683a231dc65a04da5b27_34.png)

![](img/e4521b23b2c3683a231dc65a04da5b27_35.png)

将其代回原式：
**T(n) = [T(n/4) + n/2] + n = T(n/4) + n/2 + n**

![](img/e4521b23b2c3683a231dc65a04da5b27_37.png)

![](img/e4521b23b2c3683a231dc65a04da5b27_39.png)

继续展开 **T(n/4)**：
**T(n/4) = T(n/8) + n/4**

![](img/e4521b23b2c3683a231dc65a04da5b27_40.png)

再次代回：
**T(n) = T(n/8) + n/4 + n/2 + n**

![](img/e4521b23b2c3683a231dc65a04da5b27_42.png)

![](img/e4521b23b2c3683a231dc65a04da5b27_43.png)

重复此过程 **k** 次后，我们得到：
**T(n) = T(n / 2^k) + n/2^(k-1) + n/2^(k-2) + ... + n/2 + n**

![](img/e4521b23b2c3683a231dc65a04da5b27_45.png)

![](img/e4521b23b2c3683a231dc65a04da5b27_46.png)

![](img/e4521b23b2c3683a231dc65a04da5b27_48.png)

与递归树法一样，当 **n / 2^k = 1**，即 **k = log₂ n** 时，递归停止。此时 **T(n / 2^k) = T(1)**，这是一个常数，记为 **c**。

![](img/e4521b23b2c3683a231dc65a04da5b27_50.png)

![](img/e4521b23b2c3683a231dc65a04da5b27_52.png)

因此：
**T(n) = c + n * (1 + 1/2 + 1/4 + ... + 1/2^(k-1))**

![](img/e4521b23b2c3683a231dc65a04da5b27_54.png)

![](img/e4521b23b2c3683a231dc65a04da5b27_56.png)

同样，括号内的级数和小于 **2**。所以：
**T(n) ≤ c + 2n**

![](img/e4521b23b2c3683a231dc65a04da5b27_58.png)

忽略常数项和系数，我们得到 **T(n) = O(n)**。

![](img/e4521b23b2c3683a231dc65a04da5b27_60.png)

![](img/e4521b23b2c3683a231dc65a04da5b27_61.png)

## 总结 📝

![](img/e4521b23b2c3683a231dc65a04da5b27_63.png)

![](img/e4521b23b2c3683a231dc65a04da5b27_64.png)

本节课中我们一起学习了求解递推关系 **T(n) = T(n/2) + n** 的两种核心方法。
*   使用**递归树法**，我们通过将递归过程可视化并求和每一层的成本，得出总时间复杂度为 **O(n)**。
*   使用**代入法**，我们通过反复展开递推式并求和，同样验证了结果为 **O(n)**。

![](img/e4521b23b2c3683a231dc65a04da5b27_66.png)

![](img/e4521b23b2c3683a231dc65a04da5b27_68.png)

![](img/e4521b23b2c3683a231dc65a04da5b27_70.png)

两种方法都揭示了关键点：虽然递归深度是对数级的（log n），但每一层的工作量（n, n/2, n/4...）构成一个收敛的等比数列，其总和与 **n** 同阶。因此，该算法的时间复杂度是线性的。
