## 如何编写算法 ✍️

![](img/8dd36582f66dd48f19eefcfb72127ad9_14.png)

![](img/8dd36582f66dd48f19eefcfb72127ad9_16.png)

上一节我们讨论了算法的定义，本节中我们来看看如何具体地编写一个算法。编写算法时，我们关注的是解决问题的步骤，而不是特定编程语言的语法细节。

![](img/8dd36582f66dd48f19eefcfb72127ad9_22.png)

以下是一个交换两个数字的算法示例：

![](img/8dd36582f66dd48f19eefcfb72127ad9_24.png)

![](img/8dd36582f66dd48f19eefcfb72127ad9_25.png)

![](img/8dd36582f66dd48f19eefcfb72127ad9_27.png)

```pseudocode
算法：交换两个数字
输入：两个数字 a 和 b
输出：交换后的 a 和 b

![](img/8dd36582f66dd48f19eefcfb72127ad9_29.png)

![](img/8dd36582f66dd48f19eefcfb72127ad9_30.png)

步骤：
1. 声明一个临时变量 t。
2. 将 a 的值赋给 t。
3. 将 b 的值赋给 a。
4. 将 t 的值赋给 b。
```

![](img/8dd36582f66dd48f19eefcfb72127ad9_32.png)

与C语言程序相比，算法编写有以下特点：
*   不声明参数的数据类型。数据类型是在后续编程阶段才决定的。
*   不直接声明临时变量。在算法中，我们默认可以直接使用变量，无需像在程序中那样先声明。
*   可以使用多种符号。例如，赋值操作可以用 `=`，也可以用 `:=`（源自Pascal等旧语言），甚至可以用箭头符号 `←` 来表示“将值存储到此处”。

![](img/8dd36582f66dd48f19eefcfb72127ad9_34.png)

![](img/8dd36582f66dd48f19eefcfb72127ad9_35.png)

![](img/8dd36582f66dd48f19eefcfb72127ad9_36.png)

编写算法的核心目标是清晰、易懂。只要参与项目的人员能够理解其逻辑，你可以使用任何你认为合适的语法，算法没有固定的语法格式。

## 如何分析算法 🔍

![](img/8dd36582f66dd48f19eefcfb72127ad9_45.png)

![](img/8dd36582f66dd48f19eefcfb72127ad9_47.png)

![](img/8dd36582f66dd48f19eefcfb72127ad9_48.png)

我们已经学会了如何编写算法，接下来我们探讨如何分析一个算法的优劣。分析算法主要是为了评估其效率，主要标准有以下几点：

![](img/8dd36582f66dd48f19eefcfb72127ad9_50.png)

![](img/8dd36582f66dd48f19eefcfb72127ad9_51.png)

以下是分析算法时需要考虑的主要标准：

![](img/8dd36582f66dd48f19eefcfb72127ad9_53.png)

![](img/8dd36582f66dd48f19eefcfb72127ad9_54.png)

![](img/8dd36582f66dd48f19eefcfb72127ad9_56.png)

1.  **时间效率 ⏱️**：算法执行所需的时间。我们希望算法能快速解决问题。分析后，我们通常得到一个**时间函数 `f(n)`**，它描述了时间消耗如何随输入规模 `n` 变化。
2.  **空间效率 💾**：算法运行需要消耗的内存空间。算法最终会转化为程序在机器上运行，因此了解其内存占用至关重要。
3.  **其他标准**：根据具体应用场景，可能还需要考虑：
    *   **网络数据传输量**：对于基于互联网或云的应用，算法产生的数据传输量是一个重要指标。
    *   **功耗**：对于移动设备（如笔记本电脑、平板电脑），算法的功耗是需要考虑的因素。
    *   **CPU寄存器占用**：在进行系统级编程或设备驱动开发时，算法占用的CPU寄存器数量也是一个考量因素。

![](img/8dd36582f66dd48f19eefcfb72127ad9_58.png)

![](img/8dd36582f66dd48f19eefcfb72127ad9_60.png)

具体分析哪些标准，取决于你的项目需求和目标。

## 算法分析实例 📊

让我们通过分析上面“交换两个数字”的算法，来具体看看如何进行时间和空间分析。

![](img/8dd36582f66dd48f19eefcfb72127ad9_67.png)

![](img/8dd36582f66dd48f19eefcfb72127ad9_68.png)

![](img/8dd36582f66dd48f19eefcfb72127ad9_69.png)

### 时间分析

![](img/8dd36582f66dd48f19eefcfb72127ad9_71.png)

![](img/8dd36582f66dd48f19eefcfb72127ad9_73.png)

为了分析时间，我们假设算法中的每一个简单语句（非嵌套的独立语句）执行都需要 **1个单位时间**。

![](img/8dd36582f66dd48f19eefcfb72127ad9_74.png)

![](img/8dd36582f66dd48f19eefcfb72127ad9_76.png)

![](img/8dd36582f66dd48f19eefcfb72127ad9_78.png)

对于我们的交换算法：
1.  `t = a` （赋值） -> 1个单位时间
2.  `a = b` （赋值） -> 1个单位时间
3.  `b = t` （赋值） -> 1个单位时间

![](img/8dd36582f66dd48f19eefcfb72127ad9_79.png)

![](img/8dd36582f66dd48f19eefcfb72127ad9_81.png)

![](img/8dd36582f66dd48f19eefcfb72127ad9_83.png)

因此，总时间 `f(n) = 3`。这是一个**常数**，不随输入规模 `n` 变化。

![](img/8dd36582f66dd48f19eefcfb72127ad9_85.png)

**注意**：即使某个语句很复杂（例如 `x = 5*a + 6*b`），在基础分析中，我们通常也将其视为消耗1个单位时间。当然，如果你需要进行极其详细的分析（例如分析到机器指令级别），可以拆解得更细，但这通常不是必要的。分析的详细程度取决于你的需求。

![](img/8dd36582f66dd48f19eefcfb72127ad9_87.png)

![](img/8dd36582f66dd48f19eefcfb72127ad9_88.png)

### 空间分析

![](img/8dd36582f66dd48f19eefcfb72127ad9_90.png)

空间分析主要计算算法中使用了多少变量。

![](img/8dd36582f66dd48f19eefcfb72127ad9_92.png)

对于我们的交换算法：
*   输入参数：`a`, `b` -> 2个变量
*   局部变量：`t` -> 1个变量

![](img/8dd36582f66dd48f19eefcfb72127ad9_93.png)

![](img/8dd36582f66dd48f19eefcfb72127ad9_94.png)

![](img/8dd36582f66dd48f19eefcfb72127ad9_96.png)

![](img/8dd36582f66dd48f19eefcfb72127ad9_97.png)

总共使用了 **3个变量**。我们通常说它占用了“3个字”的空间（“字”是一个与具体数据类型无关的抽象单位）。这同样是一个**常数**。

![](img/8dd36582f66dd48f19eefcfb72127ad9_99.png)

在算法复杂度分析中，我们通常用 **O(1)**（读作“大O1”）来表示这种常数级别的时间和空间复杂度。无论是3、30还是3000，只要是固定值，我们都记作 O(1)。

![](img/8dd36582f66dd48f19eefcfb72127ad9_101.png)

![](img/8dd36582f66dd48f19eefcfb72127ad9_102.png)

![](img/8dd36582f66dd48f19eefcfb72127ad9_103.png)

本节课中我们一起学习了：
1.  **编写算法**：算法关注逻辑步骤，不拘泥于编程语言语法，目标是清晰易懂。
2.  **分析算法**：主要从**时间效率**和**空间效率**两个核心维度进行评估，也可根据需求考虑网络、功耗等其他指标。
3.  **分析方法**：通过简单的“交换数字”算法实例，我们实践了基础分析：假设每个语句耗时1单位，每个变量占用1字空间，并得出其时间复杂度和空间复杂度均为 **O(1)**（常数阶）。
