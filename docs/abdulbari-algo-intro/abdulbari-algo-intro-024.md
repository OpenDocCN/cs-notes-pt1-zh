## 什么是分治函数？🔍

上一节我们介绍了递减函数，本节中我们来看看分治函数。分治函数之所以得名，是因为在算法或函数中，它调用自身时传入的参数是 **n/2**。也就是说，如果传入的参数是 `n`，那么函数会将 `n` 除以 2。

一个算法接收参数 `n` 后，可以通过多种方式减小 `n` 的值：
*   `n-1`：递减。
*   `n/2`：除以 2（分治）。
*   `√n`：开平方根。

![](img/7611969e089f9ccfb4ca89aa5925cda3_6.png)

对于这些不同的减小方式，递归函数会有不同类型的递归关系。我们已经学习过递减类型的函数，现在我们来分析分治类型的函数。

## 建立递归关系 📝

![](img/7611969e089f9ccfb4ca89aa5925cda3_10.png)

![](img/7611969e089f9ccfb4ca89aa5925cda3_12.png)

让我们来求解一个具体的分治递归函数。假设一个算法的时间函数为 `T(n)`。该算法内部包含两个部分：

1.  一个耗时 1 个时间单位的操作。
2.  一个递归调用 `T(n/2)`。

因此，我们可以建立以下递归方程：
**T(n) = T(n/2) + 1**

![](img/7611969e089f9ccfb4ca89aa5925cda3_14.png)

同时，我们需要一个基准条件。当 `n = 1` 时，函数直接返回，我们可以认为其耗时是一个常数，例如 1。所以完整的递归关系是：
*   **T(n) = T(n/2) + 1**，当 `n > 1` 时。
*   **T(1) = 1**。

![](img/7611969e089f9ccfb4ca89aa5925cda3_16.png)

接下来，我们将使用两种方法来求解这个递归式。

---

## 方法一：递归树法 🌳

递归树法通过将递归展开成一棵树来直观地计算总时间。

![](img/7611969e089f9ccfb4ca89aa5925cda3_18.png)

**步骤如下：**

![](img/7611969e089f9ccfb4ca89aa5925cda3_20.png)

![](img/7611969e089f9ccfb4ca89aa5925cda3_21.png)

1.  **展开第一层**：`T(n)` 的耗时等于 1（当前操作）加上 `T(n/2)` 的耗时。
    > T(n) = 1 + T(n/2)

2.  **展开第二层**：`T(n/2)` 又可以展开为 1 加上 `T(n/4)`。
    > T(n) = 1 + [1 + T(n/4)] = 2 + T(n/4)

![](img/7611969e089f9ccfb4ca89aa5925cda3_23.png)

![](img/7611969e089f9ccfb4ca89aa5925cda3_25.png)

3.  **继续展开**：这个过程会持续进行。假设我们展开 `k` 次，那么递归调用会变成 `T(n / 2^k)`，而前面累计的常数项之和为 `k`。
    > T(n) = k + T(n / 2^k)

![](img/7611969e089f9ccfb4ca89aa5925cda3_27.png)

4.  **确定终止条件**：递归会一直进行，直到问题规模变为 1，即 `n / 2^k = 1`。由此我们可以解出 `k`：
    > n / 2^k = 1 => n = 2^k => k = log₂n

![](img/7611969e089f9ccfb4ca89aa5925cda3_29.png)

5.  **计算总时间**：将 `k = log₂n` 和 `T(1) = 1` 代入公式：
    > T(n) = log₂n + T(1) = log₂n + 1

因此，该递归式的时间复杂度为 **O(log n)**。

![](img/7611969e089f9ccfb4ca89aa5925cda3_31.png)

![](img/7611969e089f9ccfb4ca89aa5925cda3_32.png)

## 方法二：代入法（归纳法） 🔄

![](img/7611969e089f9ccfb4ca89aa5925cda3_37.png)

上一节我们用递归树法得到了结果，本节我们用代入法来验证。代入法通过不断将递归式右侧的项用其定义进行替换来求解。

![](img/7611969e089f9ccfb4ca89aa5925cda3_39.png)

**步骤如下：**

![](img/7611969e089f9ccfb4ca89aa5925cda3_41.png)

1.  **写出原始方程**：
    > T(n) = T(n/2) + 1 ... (1)

![](img/7611969e089f9ccfb4ca89aa5925cda3_42.png)

![](img/7611969e089f9ccfb4ca89aa5925cda3_43.png)

![](img/7611969e089f9ccfb4ca89aa5925cda3_45.png)

2.  **第一次代入**：为了替换右边的 `T(n/2)`，我们写出 `T(n/2)` 的表达式：
    > T(n/2) = T(n/4) + 1
    将上式代入方程(1)：
    > T(n) = [T(n/4) + 1] + 1 = T(n/4) + 2 ... (2)

![](img/7611969e089f9ccfb4ca89aa5925cda3_47.png)

3.  **第二次代入**：继续替换 `T(n/4)`：
    > T(n/4) = T(n/8) + 1
    代入方程(2)：
    > T(n) = [T(n/8) + 1] + 2 = T(n/8) + 3 ... (3)

![](img/7611969e089f9ccfb4ca89aa5925cda3_49.png)

![](img/7611969e089f9ccfb4ca89aa5925cda3_51.png)

4.  **观察规律**：经过 `k` 次代入后，模式变得清晰：
    > T(n) = T(n / 2^k) + k ... (4)

![](img/7611969e089f9ccfb4ca89aa5925cda3_53.png)

5.  **应用基准条件**：当 `n / 2^k = 1`，即 `n = 2^k` 时，递归到达终点 `T(1)`。此时 `k = log₂n`。
    将 `k` 和 `T(1) = 1` 代入方程(4)：
    > T(n) = T(1) + log₂n = 1 + log₂n

![](img/7611969e089f9ccfb4ca89aa5925cda3_55.png)

同样，我们得出时间复杂度为 **O(log n)**。

![](img/7611969e089f9ccfb4ca89aa5925cda3_57.png)

![](img/7611969e089f9ccfb4ca89aa5925cda3_58.png)

## 总结 📖

![](img/7611969e089f9ccfb4ca89aa5925cda3_64.png)

![](img/7611969e089f9ccfb4ca89aa5925cda3_66.png)

本节课中我们一起学习了分治类型的递归关系 **T(n) = T(n/2) + 1**。

我们通过两种方法进行了求解：
*   **递归树法**：通过将递归可视化成一棵树，累加每一层的代价，最终得出时间复杂度为 **O(log n)**。
*   **代入法**：通过反复代入展开递归式，归纳出通用公式，同样验证了时间复杂度为 **O(log n)**。

![](img/7611969e089f9ccfb4ca89aa5925cda3_68.png)

这个结果非常典型：**当一个递归算法每次将问题规模减半，并执行常数时间操作时，其时间复杂度通常是对数级的 O(log n)**。理解这个推导过程是掌握更复杂递归分析（如主定理）的重要基础。在后续课程中，我们将看到更多递归式的例子并学习主定理。
