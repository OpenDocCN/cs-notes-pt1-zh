# 076：AVL树 - 插入与旋转 🎯

在本节课中，我们将要学习AVL树。AVL树是一种高度平衡的二叉搜索树，它通过旋转操作来维持树的平衡，从而保证搜索、插入和删除等操作的时间复杂度为O(log n)。我们将从二叉搜索树的基础开始，逐步理解其缺点、平衡因子的概念，以及四种核心的旋转操作，最后学习如何通过插入操作构建一个AVL树。

![](img/991d152afafa33a68c0aa8e5f4c6c636_1.png)

## 什么是二叉搜索树？ 🌲

二叉搜索树是一种特殊的二叉树结构。对于树中的任意一个节点，其左子树中的所有节点值都小于该节点的值，而其右子树中的所有节点值都大于该节点的值。

**公式**：对于任意节点 `node`，满足 `node.left.value < node.value < node.right.value`。

![](img/991d152afafa33a68c0aa8e5f4c6c636_3.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_4.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_6.png)

这种结构使得二叉搜索树非常适用于搜索操作。搜索过程从根节点开始，根据比较结果决定向左子树还是右子树移动。

![](img/991d152afafa33a68c0aa8e5f4c6c636_8.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_10.png)

### 二叉搜索树的搜索示例

![](img/991d152afafa33a68c0aa8e5f4c6c636_12.png)

假设我们有一个二叉搜索树，其节点值为 `[20, 10, 30, 5, 15, 25, 35]`。

*   **搜索成功**：查找值 `30`。
    1.  从根节点 `20` 开始，`30 > 20`，转向右子树。
    2.  到达节点 `30`，匹配成功。共进行了2次比较。
*   **搜索失败**：查找值 `32`。
    1.  从根节点 `20` 开始，`32 > 20`，转向右子树。
    2.  到达节点 `30`，`32 > 30`，转向右子树。
    3.  右子树为空，搜索失败。共进行了2次比较。

![](img/991d152afafa33a68c0aa8e5f4c6c636_14.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_15.png)

搜索所需的最大比较次数取决于树的高度。因此，二叉搜索树的搜索时间复杂度与树的高度直接相关。

![](img/991d152afafa33a68c0aa8e5f4c6c636_17.png)

## 二叉搜索树的缺点 ⚠️

![](img/991d152afafa33a68c0aa8e5f4c6c636_19.png)

上一节我们介绍了二叉搜索树的搜索效率取决于其高度。本节中我们来看看它的主要问题。

对于同一组数据，根据插入顺序的不同，可能形成高度差异巨大的二叉搜索树。树的高度最小可以是 `log₂ n`，最大可以是 `n`。

![](img/991d152afafa33a68c0aa8e5f4c6c636_21.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_22.png)

**公式**：二叉搜索树的高度 `h` 满足 `log₂ n ≤ h ≤ n`。

![](img/991d152afafa33a68c0aa8e5f4c6c636_24.png)

这意味着，在最坏情况下（例如，按升序或降序插入数据），二叉搜索树会退化成一条链表，搜索时间复杂度变为 `O(n)`，与线性搜索无异，失去了其高效搜索的优势。

![](img/991d152afafa33a68c0aa8e5f4c6c636_26.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_27.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_29.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_31.png)

### 创建二叉搜索树的示例

![](img/991d152afafa33a68c0aa8e5f4c6c636_33.png)

考虑两组相同但顺序不同的键值 `[30, 40, 10, 50, 20, 5, 35]`。

*   **顺序1**：`[30, 40, 10, 50, 20, 5, 35]`
    *   形成的树比较平衡，高度约为 `log₂ 7 ≈ 3`。
*   **顺序2**：`[50, 40, 35, 30, 20, 10, 5]`
    *   形成的树退化成右倾链表，高度为 `7`。

这个问题的根源在于，二叉搜索树的构建过程（逐个插入）无法控制最终树的高度。

## 如何改进二叉搜索树？ 💡

![](img/991d152afafa33a68c0aa8e5f4c6c636_35.png)

既然问题的核心在于树的高度不平衡，那么改进的思路就是**在插入或删除过程中，动态调整树的结构，使其保持平衡**。这就是平衡二叉搜索树的思想，AVL树是其中最早被发明的一种。

![](img/991d152afafa33a68c0aa8e5f4c6c636_37.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_38.png)

核心思想是**旋转**。通过围绕特定节点进行旋转，可以将一个不平衡的子树调整为平衡的子树。

![](img/991d152afafa33a68c0aa8e5f4c6c636_40.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_41.png)

### 旋转的基本思想

考虑三个节点 `A, B, C`。假设由于插入操作，形成了 `A(B(C))` 这种左左倾斜的不平衡结构（`C` 是 `B` 的左孩子，`B` 是 `A` 的左孩子）。

![](img/991d152afafa33a68c0aa8e5f4c6c636_43.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_44.png)

我们可以通过一次**右旋**操作来平衡它：
1.  将 `B` 提升为新的子树的根节点。
2.  `A` 变为 `B` 的右孩子。
3.  `B` 原来的右孩子（如果有）变为 `A` 的左孩子。

![](img/991d152afafa33a68c0aa8e5f4c6c636_46.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_47.png)

旋转后，结构变为 `B(C, A)`，树的高度得以降低并恢复平衡。对于右右倾斜的情况，则进行对称的**左旋**操作。

## 什么是AVL树？ ⚖️

![](img/991d152afafa33a68c0aa8e5f4c6c636_49.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_51.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_52.png)

AVL树（以发明者Adelson-Velsky和Landis命名）是一种**高度平衡的二叉搜索树**。它通过为每个节点维护一个**平衡因子**来确保树的平衡性。

![](img/991d152afafa33a68c0aa8e5f4c6c636_54.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_56.png)

**平衡因子**定义为：**一个节点的左子树高度减去其右子树高度**。

![](img/991d152afafa33a68c0aa8e5f4c6c636_58.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_60.png)

**公式**：`平衡因子(BF) = 左子树高度(h_left) - 右子树高度(h_right)`

![](img/991d152afafa33a68c0aa8e5f4c6c636_62.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_64.png)

在AVL树中，要求每个节点的平衡因子只能是 `-1`、`0` 或 `1`。如果任何节点的平衡因子绝对值大于1，则该节点不平衡，需要通过旋转来调整。

![](img/991d152afafa33a68c0aa8e5f4c6c636_66.png)

**公式**：对于AVL树中任意节点 `node`，满足 `|BF(node)| ≤ 1`。

### 平衡因子计算示例

![](img/991d152afafa33a68c0aa8e5f4c6c636_68.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_69.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_71.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_73.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_75.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_77.png)

以下是如何计算节点平衡因子的方法（高度通常定义为从节点到最深叶子节点的边数）：
*   **叶子节点**：左右子树高度均为0，平衡因子为 `0`。
*   **只有一个左孩子的节点**：左子树高度为1，右子树高度为0，平衡因子为 `1`。
*   **左子树高度为2，右子树高度为1的节点**：平衡因子为 `2 - 1 = 1`。
*   **左子树高度为3，右子树高度为1的节点**：平衡因子为 `3 - 1 = 2`（**不平衡！**）。

![](img/991d152afafa33a68c0aa8e5f4c6c636_79.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_81.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_83.png)

## AVL树的旋转操作 🔄

![](img/991d152afafa33a68c0aa8e5f4c6c636_88.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_90.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_92.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_94.png)

当插入或删除一个节点导致某个节点的平衡因子变为 `+2` 或 `-2` 时，就需要通过旋转来恢复平衡。根据不平衡的形态，有四种基本的旋转类型。

### 1. LL旋转（右单旋）

![](img/991d152afafa33a68c0aa8e5f4c6c636_96.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_97.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_99.png)

**触发条件**：在节点 `A` 的**左孩子(`B`)的左子树**上插入了新节点，导致 `A` 的平衡因子变为 `+2`。这是一种“左左”不平衡。

**操作**：以 `A` 为支点，进行一次向右的顺时针旋转。
*   `B` 成为新的子树根节点。
*   `A` 成为 `B` 的右孩子。
*   `B` 原来的右孩子 `BR` 成为 `A` 的左孩子。

![](img/991d152afafa33a68c0aa8e5f4c6c636_101.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_103.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_104.png)

**代码/图示逻辑**：
```
不平衡状态:
        A (BF=+2)
       /
      B (BF=+1)
     /
    C

![](img/991d152afafa33a68c0aa8e5f4c6c636_106.png)

执行LL旋转(右旋)后:
      B
     / \
    C   A
```

![](img/991d152afafa33a68c0aa8e5f4c6c636_108.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_109.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_111.png)

### 2. RR旋转（左单旋）

![](img/991d152afafa33a68c0aa8e5f4c6c636_112.png)

**触发条件**：在节点 `A` 的**右孩子(`B`)的右子树**上插入了新节点，导致 `A` 的平衡因子变为 `-2`。这是一种“右右”不平衡。

**操作**：以 `A` 为支点，进行一次向左的逆时针旋转。与LL旋转对称。
*   `B` 成为新的子树根节点。
*   `A` 成为 `B` 的左孩子。
*   `B` 原来的左孩子 `BL` 成为 `A` 的右孩子。

![](img/991d152afafa33a68c0aa8e5f4c6c636_114.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_115.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_117.png)

**代码/图示逻辑**：
```
不平衡状态:
    A (BF=-2)
     \
      B (BF=-1)
       \
        C

![](img/991d152afafa33a68c0aa8e5f4c6c636_119.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_121.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_122.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_123.png)

执行RR旋转(左旋)后:
      B
     / \
    A   C
```

![](img/991d152afafa33a68c0aa8e5f4c6c636_125.png)

### 3. LR旋转（左右双旋）

![](img/991d152afafa33a68c0aa8e5f4c6c636_127.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_128.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_129.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_131.png)

**触发条件**：在节点 `A` 的**左孩子(`B`)的右子树**上插入了新节点，导致 `A` 的平衡因子变为 `+2`。这是一种“左右”不平衡。

![](img/991d152afafa33a68c0aa8e5f4c6c636_133.png)

**操作**：需要两次旋转。
1.  先对 `A` 的左孩子 `B` 进行一次 **RR旋转（左旋）**，将情况转化为LL形态。
2.  再对 `A` 进行一次 **LL旋转（右旋）**。

![](img/991d152afafa33a68c0aa8e5f4c6c636_135.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_136.png)

也可以理解为：将 `B` 的右孩子 `C` 提升为新的根节点。

![](img/991d152afafa33a68c0aa8e5f4c6c636_138.png)

**代码/图示逻辑**：
```
不平衡状态:
        A (BF=+2)
       /
      B (BF=-1)
       \
        C

第一步：对B进行RR旋转(左旋)
        A
       /
      C
     /
    B

![](img/991d152afafa33a68c0aa8e5f4c6c636_140.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_141.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_142.png)

第二步：对A进行LL旋转(右旋)
        C
       / \
      B   A
```

![](img/991d152afafa33a68c0aa8e5f4c6c636_144.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_145.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_147.png)

### 4. RL旋转（右左双旋）

![](img/991d152afafa33a68c0aa8e5f4c6c636_149.png)

**触发条件**：在节点 `A` 的**右孩子(`B`)的左子树**上插入了新节点，导致 `A` 的平衡因子变为 `-2`。这是一种“右左”不平衡。

**操作**：与LR旋转对称。
1.  先对 `A` 的右孩子 `B` 进行一次 **LL旋转（右旋）**，将情况转化为RR形态。
2.  再对 `A` 进行一次 **RR旋转（左旋）**。

![](img/991d152afafa33a68c0aa8e5f4c6c636_151.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_153.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_155.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_157.png)

也可以理解为：将 `B` 的左孩子 `C` 提升为新的根节点。

**代码/图示逻辑**：
```
不平衡状态:
    A (BF=-2)
     \
      B (BF=+1)
     /
    C

第一步：对B进行LL旋转(右旋)
    A
     \
      C
       \
        B

![](img/991d152afafa33a68c0aa8e5f4c6c636_159.png)

第二步：对A进行RR旋转(左旋)
        C
       / \
      A   B
```

**总结四种旋转**：
*   **LL** 和 **RR** 是单旋。
*   **LR** 和 **RL** 是双旋，由两个单旋组合而成。
*   **核心**：无论子树多复杂，旋转操作始终只涉及三个关键节点（失衡节点、它的孩子、它的孙子），并按照上述模式重组。

![](img/991d152afafa33a68c0aa8e5f4c6c636_161.png)

## 构建AVL树：逐步插入示例 🏗️

![](img/991d152afafa33a68c0aa8e5f4c6c636_162.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_163.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_165.png)

![](img/991d152afafa33a68c0aa8e5f4c6c636_166.png)

现在，我们通过一个完整的例子，看看如何通过插入操作构建一棵AVL树。插入规则是：先像普通二叉搜索树一样插入，然后从插入点向上回溯，找到第一个不平衡的祖先节点，对其进行相应的旋转。

![](img/991d152afafa33a68c0aa8e5f4c6c636_168.png)

我们按顺序插入键值：`[40, 20, 10, 25, 30, 22, 50]`。

1.  **插入40**：树为空，直接作为根节点。平衡。
2.  **插入20**：作为40的左孩子。检查40的BF：左高1，右高0，BF=1。平衡。
    ```
        40(BF=1)
       /
      20(BF=0)
    ```
3.  **插入10**：作为20的左孩子。
    *   更新BF：20的BF变为1。40的左子树高度变为2，BF=2。**节点40不平衡！**
    *   插入位置在40的左孩子的左子树，属于 **LL型**。
    *   对40执行 **LL旋转（右旋）**。
    *   结果：20成为根，10在左，40在右。所有节点BF为0。
    ```
        20(BF=0)
       /  \
      10  40(BF=0)
    ```
4.  **插入25**：作为40的左孩子。树保持平衡。
    ```
          20(BF=-1)
         /  \
        10  40(BF=1)
            /
           25(BF=0)
    ```
5.  **插入30**：作为25的右孩子。
    *   更新BF：25的BF=-1。40的左子树高度为2，右子树高度为0，BF=2。**节点40不平衡！**
    *   插入位置在40的左孩子的右子树，属于 **LR型**。
    *   对40执行 **LR旋转**。将30提升。
    *   结果：30成为局部根，25在左，40在右。20的右孩子更新为30。
    ```
          20(BF=-1)
         /  \
        10  30(BF=0)
            /  \
           25  40(BF=0)
    ```
6.  **插入22**：作为25的左孩子。
    *   更新BF：25的BF=1。30的左子树高度为2，右子树高度为1，BF=1。40的BF=0。**节点30不平衡！**（它是第一个不平衡的祖先）
    *   插入位置在30的左孩子的左子树？不，我们需要看导致30不平衡的“三个节点”：30(失衡点)、25(孩子)、22(孙子)。22在25的左边，所以对于30-25-22这条路径，是 **左左(LL)型**吗？仔细看：新节点22插在25的左边，而25是30的左孩子，所以这是“左孩子的左子树”，属于 **LL型**。
    *   对30执行 **LL旋转（右旋）**。将25提升。
    *   结果：25成为20的右孩子，22是25的左孩子，30是25的右孩子。
    ```
          20(BF=-1)
         /  \
        10  25(BF=0)
            /  \
           22  30(BF=0)
                 \
                 40(BF=0) // 注意40现在是30的右孩子
    ```
    *   检查：现在所有节点BF绝对值均<=1，树平衡。
7.  **插入50**：作为40的右孩子。
    *   更新BF：40的BF=-1。30的BF=-1。25的右子树高度增加，BF变为-1。20的右子树高度增加，BF变为-2。**节点20不平衡！**
    *   导致20不平衡的三个节点是：20(失衡点)、25(孩子)、30(孙子)。新节点50插在30的子树中，但我们需要看20-25-30这条路径。25是20的右孩子，30是25的右孩子，所以这是“右孩子的右子树”，属于 **RR型**。
    *   对20执行 **RR旋转（左旋）**。将25提升为新的根。
    *   结果：25成为整棵树的根，20在左，30在右。20的右孩子变为22（原25的左孩子），30的左孩子保持不变（是空或某个值），右孩子是40（包含新插入的50）。
    ```
            25(BF=0)
           /   \
        20(BF=0) 30(BF=-1)
         /  \       \
        10  22      40(BF=-1)
                          \
                          50(BF=0)
    ```
    *   最终，我们得到了一棵平衡的AVL树。

![](img/991d152afafa33a68c0aa8e5f4c6c636_170.png)

**关键点**：
*   插入后必须从插入点向上回溯检查平衡因子。
*   只对**第一个**找到的不平衡祖先节点进行旋转。
*   一次正确的旋转可以使该子树恢复平衡，并可能同时解决上层的不平衡问题。

## 总结 📚

本节课中我们一起学习了AVL树，一种自平衡的二叉搜索树。

![](img/991d152afafa33a68c0aa8e5f4c6c636_175.png)

1.  **回顾了二叉搜索树**：其搜索效率取决于树的高度，但在最坏情况下会退化成链表（高度为n）。
2.  **引入了平衡的需求**：为了保持O(log n)的搜索效率，需要树的高度平衡。
3.  **定义了AVL树**：通过平衡因子（`|左子树高 - 右子树高| ≤ 1`）来严格定义平衡。
4.  **掌握了四种旋转操作**：这是维持AVL树平衡的核心技术。
    *   **LL / 右单旋**：处理“左左”不平衡。
    *   **RR / 左单旋**：处理“右右”不平衡。
    *   **LR / 左右双旋**：处理“左右”不平衡。
    *   **RL / 右左双旋**：处理“右左”不平衡。
5.  **实践了AVL树的构建**：通过逐步插入并即时旋转调整，可以构建出一棵高度平衡的AVL树。其高度h始终满足 `h = O(log n)`，具体约为 `h < 1.44 * log₂(n+2)`，从而保证了所有操作的高效性。

![](img/991d152afafa33a68c0aa8e5f4c6c636_177.png)

AVL树通过严格的平衡条件，提供了最坏情况下的性能保证。与之相比，另一种常见的平衡树——红黑树——使用稍宽松的平衡规则，减少了旋转次数，在实际应用中（如C++ STL的map/set）也非常流行。理解AVL树是学习所有平衡树结构的重要基础。
