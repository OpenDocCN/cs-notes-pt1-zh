# 061：图的遍历 - 广度优先搜索与深度优先搜索 🧭

![](img/7a209dd5421b6e8b41df275a5b01620c_1.png)

在本节课中，我们将要学习图的两种基本遍历方法：广度优先搜索和深度优先搜索。我们将通过多个例子来理解它们的工作原理、区别以及实现步骤。

## 概述

![](img/7a209dd5421b6e8b41df275a5b01620c_3.png)

图遍历是指系统地访问图中的每一个顶点。**广度优先搜索** 和 **深度优先搜索** 是两种最核心的遍历策略。理解它们的关键在于区分“访问顶点”和“探索顶点”这两个概念：访问意味着到达该顶点；探索意味着访问该顶点的所有相邻顶点。

![](img/7a209dd5421b6e8b41df275a5b01620c_5.png)

![](img/7a209dd5421b6e8b41df275a5b01620c_6.png)

![](img/7a209dd5421b6e8b41df275a5b01620c_7.png)

上一节我们介绍了图遍历的基本概念，本节中我们来看看这两种方法的具体过程。

![](img/7a209dd5421b6e8b41df275a5b01620c_9.png)

## 核心概念与快速比较

为了快速理解两者的区别，我们首先观察一个简单的图（实际上是一棵树）。

![](img/7a209dd5421b6e8b41df275a5b01620c_1.png)

### 广度优先搜索示例

1.  选择顶点1作为起始顶点。
2.  **访问** 顶点1。
3.  开始**探索**顶点1，即访问其所有相邻顶点（2和4）。访问顺序可以是任意的，例如先2后4。
4.  接着，选择下一个待探索的顶点。在已访问的顶点（1, 2, 4）中，按照访问顺序，选择顶点2进行探索。
5.  探索顶点2，访问其所有未访问的相邻顶点（3, 6, 7）。
6.  以此类推，直到所有顶点都被访问。

![](img/7a209dd5421b6e8b41df275a5b01620c_12.png)

![](img/7a209dd5421b6e8b41df275a5b01620c_13.png)

### 深度优先搜索示例

![](img/7a209dd5421b6e8b41df275a5b01620c_16.png)

1.  同样从顶点1开始，**访问**顶点1。
2.  开始**探索**顶点1，首先访问相邻顶点2。
3.  **关键规则**：一旦访问了一个新顶点（2），就**暂停**当前顶点（1）的探索，立即开始**探索**这个新顶点（2）。
4.  探索顶点2，访问其一个相邻顶点3。
5.  再次应用规则：暂停顶点2的探索，开始探索新顶点3。
6.  顶点3没有未访问的相邻顶点，因此顶点3探索完成。**回溯**到上一个顶点（2），继续探索它的其他相邻顶点（6, 7）。
7.  此过程持续进行，直到所有顶点都被探索。

![](img/7a209dd5421b6e8b41df275a5b01620c_18.png)

遍历顺序可能是：**1 -> 2 -> 3 -> 6 -> 7 -> 4**。

### 核心区别

![](img/7a209dd5421b6e8b41df275a5b01620c_22.png)

*   **广度优先搜索**：探索一个顶点时，会先访问其**所有**相邻顶点，然后再选择下一个顶点进行探索。这类似于树的**层序遍历**。
*   **深度优先搜索**：探索一个顶点时，访问一个相邻顶点后，会立即深入探索这个新顶点，直到无路可走再回溯。这类似于树的**先序遍历**。

![](img/7a209dd5421b6e8b41df275a5b01620c_24.png)

### 算法步骤

![](img/7a209dd5421b6e8b41df275a5b01620c_33.png)

![](img/7a209dd5421b6e8b41df275a5b01620c_35.png)

以下是BFS的步骤分解：

![](img/7a209dd5421b6e8b41df275a5b01620c_37.png)

![](img/7a209dd5421b6e8b41df275a5b01620c_39.png)

1.  **初始化**：选择任意一个顶点作为起点（例如顶点1）。将其标记为已访问，加入结果列表，并**入队**。
    `Q = [1]`, `Result = [1]`

![](img/7a209dd5421b6e8b41df275a5b01620c_41.png)

![](img/7a209dd5421b6e8b41df275a5b01620c_42.png)

![](img/7a209dd5421b6e8b41df275a5b01620c_44.png)

![](img/7a209dd5421b6e8b41df275a5b01620c_46.png)

2.  **重复以下步骤，直到队列为空**：
    a.  **出队**一个顶点（记为 `current`）。
    b.  **探索**顶点 `current`：遍历它的每一个相邻顶点。
    c.  对于每个相邻顶点，如果它**未被访问过**，则：
        *   标记为已访问。
        *   加入结果列表。
        *   **入队**。

![](img/7a209dd5421b6e8b41df275a5b01620c_48.png)

### 分步演示

假设我们从顶点1开始。

*   **初始**：`Q = [1]`, `Result = [1]`
*   **迭代1**：出队 `1`。探索1，相邻顶点为4和2。两者均未访问，按顺序访问4，然后2。将4和2入队。
    `Q = [4, 2]`, `Result = [1, 4, 2]`
    ![](img/7a209dd5421b6e8b41df275a5b01620c_26.png)
*   **迭代2**：出队 `4`。探索4，相邻顶点为3。访问3并入队。
    `Q = [2, 3]`, `Result = [1, 4, 2, 3]`
    ![](img/7a209dd5421b6e8b41df275a5b01620c_27.png)
*   **迭代3**：出队 `2`。探索2，相邻顶点为3, 5, 7, 8。3已访问，依次访问5, 8, 7并入队。
    `Q = [3, 5, 8, 7]`, `Result = [1, 4, 2, 3, 5, 8, 7]`
    ![](img/7a209dd5421b6e8b41df275a5b01620c_29.png)
    ![](img/7a209dd5421b6e8b41df275a5b01620c_31.png)
*   **迭代4**：出队 `3`。探索3，相邻顶点为2, 8, 9, 10。2和8已访问，访问10和9并入队。
    `Q = [5, 8, 7, 10, 9]`, `Result = [1, 4, 2, 3, 5, 8, 7, 10, 9]`
    ![](img/7a209dd5421b6e8b41df275a5b01620c_35.png)
*   **迭代5**：出队 `5`。探索5，相邻顶点为6, 7, 8。7和8已访问，访问6并入队。
    `Q = [8, 7, 10, 9, 6]`, `Result = [1, 4, 2, 3, 5, 8, 7, 10, 9, 6]`
    ![](img/7a209dd5421b6e8b41df275a5b01620c_41.png)
*   后续迭代中，出队8, 7, 10, 9, 6，但它们的相邻顶点均已访问，不再有新顶点入队。队列最终为空，算法结束。

![](img/7a209dd5421b6e8b41df275a5b01620c_50.png)

![](img/7a209dd5421b6e8b41df275a5b01620c_52.png)

最终遍历顺序为：**1, 4, 2, 3, 5, 8, 7, 10, 9, 6**。

### BFS要点总结

![](img/7a209dd5421b6e8b41df275a5b01620c_57.png)

1.  **起点自由**：可以从任何顶点开始。
2.  **访问顺序自由**：探索一个顶点时，其相邻顶点的访问顺序可以任意。
3.  **核心规则**：必须**完全探索**完一个顶点（访问其所有相邻顶点）后，才能从队列中取出下一个顶点进行探索。
4.  **数据结构**：使用**队列**管理待探索的顶点。
5.  **生成树**：BFS遍历过程中经过的边构成一棵**BFS生成树**，未包含的边称为**交叉边**。
6.  **多解性**：由于起点和访问顺序的自由度，一个图可以有多个有效的BFS序列。
    *   例如：`1, 2, 4, 8, 5, 7, 3, 6, 10, 9`
    *   例如：`5, 8, 7, 6, 2, 3, 1, 4, 10, 9`
    ![](img/7a209dd5421b6e8b41df275a5b01620c_57.png)
    ![](img/7a209dd5421b6e8b41df275a5b01620c_59.png)

## 深度优先搜索详解 🟩

我们将使用一个栈来辅助实现DFS（递归本质也是栈）。

![](img/7a209dd5421b6e8b41df275a5b01620c_59.png)

![](img/7a209dd5421b6e8b41df275a5b01620c_61.png)

### 算法步骤

![](img/7a209dd5421b6e8b41df275a5b01620c_61.png)

以下是DFS的步骤分解：

![](img/7a209dd5421b6e8b41df275a5b01620c_63.png)

1.  **初始化**：选择任意一个顶点作为起点（例如顶点1）。将其标记为已访问，加入结果列表，并**压栈**。
    `Stack = [1]`, `Result = [1]`

2.  **重复以下步骤，直到栈为空**：
    a.  查看**栈顶**顶点（记为 `current`），但不弹出。
    b.  从 `current` 的相邻顶点中，找到一个**未被访问**的顶点（记为 `next`）。
    c.  如果找到 `next`：
        *   标记 `next` 为已访问，加入结果列表。
        *   将 `next` **压栈**。（此时 `next` 成为新的 `current`，算法将在下一步探索它）。
    d.  如果没找到（即 `current` 的所有相邻顶点都已访问）：
        *   将 `current` **弹出栈**。算法将回溯到上一个顶点继续探索。

### 分步演示（思路）

![](img/7a209dd5421b6e8b41df275a5b01620c_65.png)

![](img/7a209dd5421b6e8b41df275a5b01620c_66.png)

假设我们从顶点1开始。

1.  访问1，压栈。`Stack = [1]`, `Result = [1]`
2.  栈顶是1。找到未访问的相邻顶点4。访问4，压栈。`Stack = [1, 4]`, `Result = [1, 4]`
3.  栈顶是4。找到未访问的相邻顶点3。访问3，压栈。`Stack = [1, 4, 3]`, `Result = [1, 4, 3]`
4.  栈顶是3。找到未访问的相邻顶点10。访问10，压栈。`Stack = [1, 4, 3, 10]`, `Result = [1, 4, 3, 10]`
5.  栈顶是10。没有未访问的相邻顶点。**弹出10**。`Stack = [1, 4, 3]`
6.  栈顶回到3。找到下一个未访问的相邻顶点9。访问9，压栈... 此过程持续进行，不断深入，直到无法继续则回溯。

![](img/7a209dd5421b6e8b41df275a5b01620c_65.png)
![](img/7a209dd5421b6e8b41df275a5b01620c_66.png)

### DFS要点总结

![](img/7a209dd5421b6e8b41df275a5b01620c_74.png)

![](img/7a209dd5421b6e8b41df275a5b01620c_76.png)

![](img/7a209dd5421b6e8b41df275a5b01620c_78.png)

1.  **起点自由**：可以从任何顶点开始。
2.  **深入优先**：一旦访问一个新顶点，就立即暂停当前路径，转而深入探索这个新顶点。
3.  **核心规则**：使用**栈**来记录访问路径，以便回溯。
4.  **生成树**：DFS遍历过程中经过的边构成一棵**DFS生成树**，未包含的边称为**回边**。
5.  **多解性**：由于起点和每次选择下一个顶点的自由度，一个图可以有多个有效的DFS序列。
    *   例如：`1, 2, 8, 7, 5, 6, 3, 9, 10, 4`
    *   例如：`3, 4, 1, 2, 5, 6, 7, 8, 10, 9`
    ![](img/7a209dd5421b6e8b41df275a5b01620c_78.png)
    ![](img/7a209dd5421b6e8b41df275a5b01620c_80.png)

## 总结

![](img/7a209dd5421b6e8b41df275a5b01620c_81.png)

本节课中我们一起学习了图的两种基础遍历算法：

*   **广度优先搜索**：采用“广撒网”的策略，使用**队列**，按层次遍历顶点。其时间复杂度为 **O(V)**。
*   **深度优先搜索**：采用“一条路走到底”的策略，使用**栈**（或递归），沿着分支深入探索。其时间复杂度也为 **O(V)**。

两者都是系统访问图中所有顶点的方法，BFS常用于寻找最短路径，DFS则更适合拓扑排序、检测环等场景。理解它们的工作机制是学习更复杂图算法的重要基础。
